# 问题追问知识包

这个文件是 `lyt-problem-clarifier` 的随包知识来源。它用于把模糊经营问题追问成可诊断问题。它必须能随 skill 一起发布，不能依赖维护者本地路径。

使用时不要整段照搬。根据用户问题类型和已提供附件选择必要问题。

## 追问总原则

用户说「为什么不出单」「为什么没流量」「为什么广告跑不出去」时，他说的是结果，不是原因。

检查顺序：

1. 先扫概念、隐含假设、因果关系和事实前提。
2. 只输出最前面的阻断层。
3. 前四层没有问题时，再界定对象、时间段、成交场景和当前结果。
4. 最后问最近动作、数据断点和最小证据。

不要一开始就解释原因，也不要直接给优化动作。追问 skill 的目标是把问题问清楚，不是把问题诊断完。

## 五层诊断漏斗

| 层级 | 核心问题 | 通过标准 |
| --- | --- | --- |
| 概念界定 | 关键名词能否对应到可观察现象 | 能区分 0 曝光、低曝光、有曝光无点击等状态 |
| 隐含假设 | 问题预设的前提是否已被证明 | 不把集中度、单个案例或经验判断直接当市场结论 |
| 因果关系 | A 和 B 是否真有因果，而非同时发生 | 有时间顺序、对照或机制证据 |
| 事实前提 | 数据、状态、平台规则和周期是否确认 | 口径明确，时效性信息已核查 |
| 信息充分性 | 证据是否能区分至少两种解释 | 有一个能改变判断的关键事实或数字 |

每轮优先处理一个阻断层。问题库里的问题只是备选，只选择当前证据无法回答的部分。

## 界定问题

不要接受过于宽泛的问题，也不要把问题问到过窄。要把问题界定成具体、可操作的版本。

| 用户原问题 | 问题太宽泛在哪里 | 应该界定成 |
| --- | --- | --- |
| 为什么不出单？ | 不知道对象、时间段、成交场景和断点 | 哪个商品/店铺，在什么时间段，从哪个流量入口开始不出单，曝光、点击、加购、下单哪个先掉 |
| 为什么没流量？ | 不知道是 0 曝光、低曝光，还是有曝光没人点 | 是商品不能展示、排名太低、标题搜不到，还是点击反馈差 |
| 为什么广告跑不出去？ | 不知道广告类型、已跑时长和实际消耗 | 默认 GMV Max，先看已跑多久、预算消耗率、实际消耗、曝光、点击、产品和素材 |
| 为什么视频爆了不出单？ | 播放量和购买意愿混在一起 | 视频播放到商品页点击、点击到加购、加购到下单，哪个环节断 |

界定模板：

```text
我先把问题界定一下：你现在不是在问「为什么不出单」这个大问题，而是在问「{对象} 在 {时间段} 通过 {成交场景} 获得了 {已知数据}，但卡在 {未知断点}」。

要继续判断，先补这几个最关键的信息。
```

## 成交链路和成交场景

总链路：

```text
商品/账号资格 -> 曝光 -> 点击 -> 加购/咨询 -> 下单 -> 成交 -> 履约/评价 -> 利润
```

GMV 公式：

```text
GMV = 曝光 × 点击率 × 点击成交率 × 客单价
```

成交不是单点动作，而是一组变量共同作用。成交成立至少要看：

| 变量 | 含义 | 追问方向 |
| --- | --- | --- |
| 需求 | 用户是否真的需要 | 产品解决什么问题，目标人群是否准确 |
| 购买力 | 用户是否买得起 | 售价、运费、同类竞品价格、客单价 |
| 信任 | 用户是否相信这个链接和商品 | 销量、评价、详情页、物流、售后、达人/内容背书 |
| 购买理由 | 用户为什么现在买 | 主图、标题、素材、场景、优惠、活动 |
| 不买理由 | 用户为什么犹豫 | 价格高、物流慢、评价少、详情不清、规格不明、内容和商品不匹配 |

不同成交场景追问不同证据：

| 场景 | 成交路径 | 最先问什么 |
| --- | --- | --- |
| 商品卡/搜索 | 搜索关键词 -> 曝光 -> 点击 -> 加购 -> 下单 -> 成交 | 标题关键词、商品状态、曝光、点击、点击率、价格、评价、物流 |
| 短视频/推荐 | 推荐 -> 观看 -> 点击小黄车 -> 加购 -> 下单 -> 成交 | 播放、完播/互动、视频到商品页点击率、评论问题、素材和商品是否匹配 |
| 直播 | 推荐 -> 进房 -> 停留 -> 点击商品 -> 下单 -> 成交 | 进房、停留、点击商品、讲品话术、优惠、库存、下单 |
| GMV Max | 商品基础盘/素材 -> 系统调度 -> 曝光 -> 点击 -> 加购 -> 下单 -> 成交 -> ROI | 已跑时长、预算、投入产出比目标、实际消耗、曝光、点击、产品数、素材、链接历史数据 |

## 通用开场

用户只给一句模糊问题时：

```text
这个问题现在不能直接判断原因，我先帮你把断点追出来。

先回答 3 个问题就行：

1. 这次问题对应哪个对象：店铺、某个商品链接、某条视频、直播间，还是 GMV Max 广告？
2. 问题从什么时候开始，最近 3-7 天做过哪些改动？
3. 现在卡在哪一步：没曝光、有曝光没点击、有点击没加购、有加购但没下单，还是有下单但利润差？

你可以直接按 1-3 简单回答，不用整理成表格。
```

如果用户已经给出部分数据，先复述：

```text
已知信息：{用户已给的信息}。
现在缺的是：{能推进判断的最小缺口}。
```

## 不出单

不要先判断商品不行。不出单可能是没有曝光、点击弱、加购弱、下单弱、信任弱、价格弱、物流弱，也可能只是样本太小。

第一轮追问：

```text
先不要急着判断商品不行。你先补 4 个信息：

1. 这是哪个对象不出单：整个店铺、某个商品、某条视频，还是某个广告？
2. 之前能出单吗？最近一次出单是哪天？从哪天开始不出了？
3. 不出单之前，你做过什么改动？改价、改图、标题、库存、优惠、广告、素材、达人、物流、活动，有没有动过？
4. 最近 3-7 天的曝光、点击、点击率、加购、下单、广告消耗、成交额分别是多少？有多少给多少。
```

用户已经说「不出单 / 没成交」时：

- 成交默认按 0 看。
- 不要再问「有没有成交」。
- 继续问成交前信号。

二轮追问：

```text
你已经明确说不出单，所以成交先按 0 看。现在要看成交前有没有信号：

1. 点击之后有没有加购？有几次？
2. 有没有下单、咨询/私信，或者用户在评论区问价格/怎么买？
3. 这个数据周期是 1 天、3 天、7 天，还是更久？
4. 售价、运费、配送时效、评价数量，和同类竞品比有没有明显劣势？
```

注意样本：

- 点击少于 50，不要因为 0 成交就判断商品、详情页或人群一定有问题。
- 曝光少于 500，只能初步看入口信号，不适合下大结论。
- 先问数据周期，避免把一天的小波动当成长期问题。

## 上品没流量 / 0 曝光

先判断「不能展示」还是「展示后反馈差」。

```text
上品没流量，先判断它是「不能展示」还是「能展示但没人点」。

你先补 4 个信息：

1. 这个商品什么时候上架？现在后台商品状态是可售，还是审核中、拒审、暂停？
2. 后台有没有商品信息质量、提升商品信息质量、账号健康、商品诊断之类提示？
3. 上架后有没有改过类目、属性、标题、图片、品牌、物流模板、库存？
4. 现在累计曝光和点击是多少？是完全 0 曝光，还是有少量曝光但没人点？
```

如果是完全 0 曝光：

- 优先问商品状态、类目、属性、标题、库存、物流模板、账号健康、商品诊断。
- 不要直接让用户改主图。
- 如果用户说后台有质量提示，先让他把提示内容贴出来。

如果是有曝光但点击低，进入商品卡点击问题。

## 商品卡有曝光但点击少

商品卡是搜索流量。用户已经有明确需求，关键是能不能看到你、愿不愿点你。

商品卡链路：

```text
搜索关键词 -> 曝光 -> 点击 -> 加购 -> 下单 -> 成交
```

追问模板：

```text
如果是商品卡有曝光但点击少，先别改一堆东西。你先补 4 个信息：

1. 用户主要通过哪些关键词搜到这个商品？你的标题前半段有没有覆盖这些核心词？
2. 这个商品最近 3-7 天曝光、点击、点击率是多少？
3. 你的主图、价格、销量/评价，和同一搜索结果页里的竞品相比有没有明显劣势？
4. 商品是新链接还是老链接？之前有没有销量、评价和自然点击数据？
```

判断方向：

| 数据现象 | 先问什么 |
| --- | --- |
| 0 曝光 | 商品状态、标题关键词、类目属性、库存物流、账号/商品诊断 |
| 有曝光没点击 | 主图、标题前半段、价格、销量评价、搜索词匹配 |
| 有点击没加购 | 详情页、价格、规格、评价、物流、购买理由 |
| 有加购没下单 | 优惠、包邮、物流时效、信任、价格对比 |

## 有点击没成交

有点击只说明用户愿意看，不代表用户愿意买。成交要看购买理由是否大于不买理由。

追问模板：

```text
有点击没成交，先看样本够不够，再看成交前信号。

你先补 4 个信息：

1. 最近这个链接有多少点击？少于 50、50-100，还是 100 以上？
2. 点击之后有没有加购、下单、咨询/私信？分别是多少？
3. 最近有没有改价格、优惠、运费、规格、详情页、评价展示、配送时效？
4. 竞品价格、评价数量、物流时效大概是什么水平？你的链接有没有明显劣势？
```

如果点击很多但没有加购，优先问：

- 详情页是否讲清楚产品用途、规格、尺寸、材质和使用场景。
- 价格是否比竞品明显更高。
- 评价、销量、售后、物流是否不足。

如果有加购但没下单，优先问：

- 优惠、包邮、物流时效、库存、规格、价格差、信任因素。

## 爆款视频不出单

爆款视频不出单，不一定是详情页问题。播放量高只说明内容有传播性，不等于目标用户有购买意愿。

先拆三条链：

| 链路 | 成立条件 |
| --- | --- |
| 算法链 | 视频能被看完、互动、传播 |
| 种草链 | 看视频的人产生购买兴趣 |
| 成交链 | 点击商品后相信商品、价格、物流和链接 |

追问模板：

```text
爆款视频不出单，先不要直接判断详情页差。你先补 5 个信息：

1. 这条视频播放量、完播率/互动率、点击商品人数分别是多少？
2. 视频到商品页点击率是多少？如果没有点击率，就给播放和商品点击。
3. 点击之后有多少加购、下单、成交？
4. 评论区用户主要在问什么？是问「多少钱/怎么买」，还是问「这是什么/有什么用」？
5. 视频内容和商品卖点是否一致？比如视频吸引的是泛娱乐人群，还是明确有购买需求的人群？
```

判断方向：

| 现象 | 追问重点 |
| --- | --- |
| 播放高，商品点击少 | 泛流量、内容和商品不匹配、引导点击弱 |
| 商品点击多，加购少 | 产品吸引力、价格、详情页、评价、物流 |
| 加购多，下单少 | 信任、优惠、价格、包邮、物流时效 |
| 评论问「这是什么」 | 内容没有把产品用途讲清楚 |
| 评论问「多少钱/怎么买」 | 有需求，重点查信任和购买路径 |

不要因为视频爆了就要求继续追爆款。先看它是否带来目标用户和购买信号。

## 广告跑不出去 / GMV Max 低消耗

默认先按 TikTok Shop 商品成交额最大化广告，也就是 GMV Max 处理。

如果用户没有主动提异常提示，默认正常投放中，不问广告状态。

追问模板：

```text
广告跑不出去或低消耗，默认先按 TikTok Shop 商品成交额最大化广告，也就是 GMV Max 处理；如果你没有提到异常提示，我先按正常投放中看。

你先补 5 个信息：

1. 广告建了多久？从几点开始跑，到现在实际跑了多长时间？
2. 当前日预算、实际消耗、曝光、点击分别是多少？
3. 投入产出比目标是多少？大概毛利率是多少，保本投入产出比有没有算过？
4. 选了几个产品？这些产品是新链接还是有历史成交/加购/点击？
5. 有没有可用视频素材？如果没有视频素材，商品卡自然流量有没有曝光、点击、加购、下单？
```

GMV Max 规则：

- 如果用户没有明确说明广告类型，默认他开的是 TikTok Shop 商品成交额最大化广告。
- 不按普通广告追问投放目标、出价方式、人群、地区定向。
- 核心可控项是预算、投入产出比目标、产品选择和素材选择。
- 用户已经明确没视频素材时，不要再问系统自动选素材。
- 只有用户主动说后台显示审核中、受限、拒审或其他异常提示时，才优先查审核和账户/商品资格。

预算消耗率：

```text
预算消耗率 = 实际消耗 / 日预算
```

| 预算消耗率 | 追问方向 |
| --- | --- |
| 低于 50% | 跑不出去，先问投入产出比目标、商品基础盘、素材、链接历史数据 |
| 50%-80% | 有探索机会，问是否有订单、点击成本、产品集中度 |
| 高于 80% | 接近预算上限，问 ROI、订单稳定性、保本投入产出比 |

如果用户已经明确「没素材、商品卡没有流量、只选 1 个新品、24 小时无消耗」，追问顺序：

1. 商品卡/搜索入口：标题有没有覆盖买家会搜的核心词，商品卡自然曝光是否为 0。
2. 商品历史信号：新品、0 成交、0 加购、0 自然点击，系统没有历史反馈。
3. 投入产出比目标是否偏紧；不要把「降低投入产出比目标」写成唯一动作。
4. 素材缺口；没有视频素材时，补素材是增加内容入口，但当前 0 消耗还要先看商品卡搜索入口。

eCPM 背景只用于追问，不要长篇解释：

```text
eCPM = 出价 × 预估点击率 × 预估转化率 × 1000
```

GMV Max 里投入产出比目标会影响系统愿意为成交付出的成本；链接历史点击和成交会影响系统对点击率、转化率的预估。

## 突然下滑

突然下滑，一定先找变化点，不要直接说平台限流。

```text
突然下滑，先找变化点。

你先补 4 个信息：

1. 下滑从哪天开始？是曝光先掉、点击先掉、加购先掉、下单先掉，还是广告消耗先掉？
2. 下滑前 3 天你做过什么动作？改商品、改广告、改预算、改投入产出比目标、改素材、改库存、报活动、换达人、断货、改物流，有没有？
3. 同期有没有活动结束、竞品降价、评价变差、物流变慢、账号/商品提示？
4. 给我下滑前后各 3 天的曝光、点击、加购、下单、成交额、广告消耗。
```

如果用户只说「突然没单」，优先问：

- 曝光有没有掉。
- 点击率有没有掉。
- 加购/下单有没有掉。
- 广告消耗有没有掉。
- 最近动作是否影响链接权重、库存、价格、物流或素材。

## 投入产出比差

投入产出比差，不等于广告一定差。要先看真实利润和订单稳定性。

追问模板：

```text
投入产出比差，先别只看后台一个数。你先补 5 个信息：

1. 这是 GMV Max、商品广告，还是自然流量整体投入产出比？
2. 这段时间广告消耗、成交额、订单数、平均下单成本分别是多少？
3. 商品毛利率大概多少？保本投入产出比有没有算过？
4. 订单是不是集中在少数产品，还是很多产品都有消耗但没订单？
5. 最近退款、差评、物流问题有没有上升？
```

判断方向：

| 现象 | 追问重点 |
| --- | --- |
| 消耗低，投入产出比高 | 样本太少，问预算消耗率、订单是否稳定 |
| 消耗高，投入产出比低 | 承接、价格、素材、人群、毛利 |
| 投入产出比高但成交额小 | 探索机会少，问预算、素材、产品数、目标是否太紧 |
| 成交额高但利润薄 | 毛利、退款、物流、佣金、广告真实成本 |
| 单子集中少数产品 | 产品分组、是否需要拆计划或单独观察 |

## 最小证据表

| 问题类型 | 最少要问到什么 |
| --- | --- |
| 不出单 | 对象、时间段、最近动作、曝光、点击、加购、下单、广告消耗 |
| 没流量 / 0 曝光 | 商品状态、后台提示、类目/属性/标题/库存/物流、累计曝光点击 |
| 商品卡点击低 | 核心搜索词、标题、曝光、点击率、主图/价格/评价对比 |
| 有点击没成交 | 点击量、加购、下单、数据周期、价格/评价/物流对比 |
| 爆款视频不出单 | 播放、商品点击、加购、下单、评论问题、内容和商品匹配度 |
| GMV Max 低消耗 | 已跑时长、预算、消耗、曝光点击、投入产出比目标、产品数、素材、链接历史数据 |
| 突然下滑 | 下滑开始日期、哪个指标先掉、下滑前动作、前后 3 天数据 |
| 投入产出比差 | 消耗、成交额、订单数、平均下单成本、毛利率、保本投入产出比、退款差评 |

## 追问组织原则

- 用户只给一句话：先指出最前面的阻断层，再问必要问题。
- 用户已经给数据：先复述已知信息，再问缺口。
- 用户已明确没成交：成交默认 0，问成交前信号。
- 用户说广告低消耗：默认 GMV Max，问已跑时长、实际消耗、曝光、点击、预算、投入产出比目标、产品、素材、链接历史数据。
- 用户说爆款视频不出单：先问视频到商品页点击率，不要直接归因到详情页。
- 用户说没流量：先判断 0 曝光还是有曝光没点击。
- 附件已经提供的信息不要重复追问；剩余问题按阻断层分轮提出。
- 只要信息足够进入诊断，就停止追问，提示可以进入数据分析或问题定位。
