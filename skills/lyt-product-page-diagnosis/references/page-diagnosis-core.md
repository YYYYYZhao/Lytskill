# 商品页诊断核心规则

本文件是商品页诊断的必读核心。它只定义诊断边界、购买链路、证据强度和常见误判；具体主图、详情页、归因和改版方案读取对应小文件。

## 购买链路

商品页诊断按这条链路拆：

```text
曝光 -> 点击 -> 进入详情页 -> 理解产品 -> 产生购买理由 -> 建立信任 -> 消除疑虑 -> 加购/下单
```

不同页面位置负责不同任务：

| 位置 | 首要任务 | 常见指标 |
| --- | --- | --- |
| 商品卡 / 主图 | 让用户一眼看懂并愿意点进来 | 曝光、点击、点击率 |
| 商品首屏 | 承接点击预期，确认这是用户想看的商品 | 点击后停留、跳失、加购前行为 |
| 详情页 | 补足购买理由、规格、证据和风险解释 | 加购、下单、点击到下单转化 |
| SKU / 价格 / 运费 | 降低决策阻力，避免预期错配 | 加购率、下单率、退款与差评线索 |

不要把所有问题都归到图片。价格、评价、销量、标题、流量人群、物流、合规和商品需求都可能影响结果。

## 证据强度

| 证据状态 | 能判断 | 不能判断 |
| --- | --- | --- |
| 只有图片 | 可见信息、理解成本、误导风险 | 点击率、转化率、主要卡点 |
| 图片 + 业务背景 | 页面表达是否匹配假设人群 | 页面是否导致指标变化 |
| 图片 + 单期指标 | 当前链路哪段弱 | 弱项一定由页面造成 |
| 修改前后 + 样本量足够 | 改版可能影响了哪个指标 | 没有控制变量时仍不能强因果 |
| 多商品/多版本/多渠道对照 | 相对更可信的页面假设 | 平台绝对因果证明 |

结论用「高 / 中 / 低置信度」。证据不足时，先降低结论强度，而不是强行给答案。

## 图片取证顺序

收到图片先记录：

1. 图片类型：主图、副图、详情页、商品首屏、后台截图、竞品页面。
2. 完整性：是否裁切、模糊、遮挡、压缩、缺少上下文。
3. 商品识别：产品是什么、数量/规格/颜色/套装是否看得清。
4. 文案识别：标题、卖点、促销、功效、材质、尺寸、认证、注意事项。
5. 场景识别：谁在什么场景使用，是否真实可信。
6. 风险识别：夸大、误导、侵权、合规、预期错配、规格冲突。

图片中看不到的内容写「未见」，不要用常识补齐。

## 最小追问原则

只问会改变判断的问题。

优先问：

- 国家/平台
- 商品是否已上线
- 统计时间
- 用户要判断点击、加购、下单还是上线前风险
- 与目标相关的最小指标
- 是否有修改前后版本或同批对照

不要一次性索取完整店铺数据，除非用户明确要做完整经营复盘；那应转 `lyt-data-analysis`。

## 判断语气

推荐表达：

- 「图片能证明的是……」
- 「当前只能判断到……」
- 「要把它上升到点击归因，还缺……」
- 「这个问题更像信息风险，不是已经确认的经营卡点」
- 「这是假设，不是事实」

避免表达：

- 「主图一定拖累点击」
- 「详情页是最大阻断」
- 「改这个转化就会提升」
- 「竞品这样做，所以你也照抄」

## 常见误判

1. 只看图片风格，忽略商品是否被看懂。
2. 把「不高级」等同于「卖不动」。
3. 把 0 下单直接归因到详情页，但样本点击很少。
4. 看到竞品销量高，就认为竞品主图一定好。
5. 看到自己主图更好看，就认为一定能赢。
6. 多个变量同时改，还把结果归到某一个变量。
7. 忽略价格、评价、运费、物流、SKU 和流量来源。
8. 把上线前信息风险检查包装成经营归因。
9. 把审美偏好当成目标用户偏好。
10. 没有合规依据就写功效、认证或夸张效果。
