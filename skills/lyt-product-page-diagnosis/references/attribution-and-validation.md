# 页面归因与验证

本文件用于有经营数据、改版前后数据、对照组或用户要求判断页面是否影响点击、加购、下单的场景。

## 指标口径

先校验分母：

| 指标 | 常见公式 | 诊断意义 |
| --- | --- | --- |
| 点击率 | 点击量 / 曝光量 | 商品卡、主图、标题、价格和评价共同影响 |
| 加购率 | 加购量 / 点击量或访问量 | 详情页、价格、SKU 和购买理由共同影响 |
| 下单率 | 下单量 / 点击量、访问量或加购量 | 详情页、价格、物流、信任和支付共同影响 |
| 点击到下单转化率 | 下单量 / 点击量 | 点击后整体承接能力 |

用户给出的平台原口径优先保留，不明口径时不要自行换算成事实。

## 样本护栏

- 曝光太少，不能判断点击率。
- 点击少于 50，不能因为 0 下单判断详情页是主要卡点。
- 只有一天数据，先看是否有异常流量、活动、断货、改价或系统波动。
- 不同流量来源混在一起时，不能直接比较主图或详情页效果。
- 多个变量同时变化时，只能列竞争假设，不能单点归因。

样本不足时，输出「当前只能判断到哪一步」，不要硬给 P0/P1/P2。

## 归因强度

| 归因强度 | 条件 | 可输出 |
| --- | --- | --- |
| 弱 | 只有图片或单期数据 | 页面风险、待验证假设 |
| 中 | 有页面 + 同期链路数据 | 哪段链路更可疑，仍保留竞争假设 |
| 强 | 有修改前后、样本量和相对单一变量 | 改版可能影响的指标与下一轮验证 |
| 很强 | 有 A/B、同批对照或稳定多周期复现 | 更可信的页面影响判断 |

即使强归因，也避免写成平台绝对因果。

## 主图归因

判断主图是否影响点击，需要：

1. 曝光量和点击量。
2. 商品卡展示中用户实际看到的主图、标题、价格、销量、评价。
3. 流量来源和统计时间。
4. 修改前后或同批对照。

可能的竞争假设：

- 价格高于同屏竞品。
- 标题关键词不匹配搜索意图。
- 评价、销量或店铺信任弱。
- 曝光人群不准。
- 商品需求弱，主图只是暴露问题。

## 详情页归因

判断详情页是否影响加购或下单，需要：

1. 点击量或访问量。
2. 加购量、下单量和对应分母。
3. 价格、优惠、SKU、运费、物流、评价、售后条件。
4. 详情页版本和统计时间。
5. 可用对照。

可能的竞争假设：

- 点击来自低意向流量。
- 价格或运费造成犹豫。
- SKU 选择复杂。
- 评价、销量或信任不足。
- 商品需求被主图吸引但详情页无法承接。

## 优先级

只有证据足够时才使用 P0/P1/P2：

- P0：信息错误、规格冲突、误导、合规风险、严重看不懂。
- P1：影响点击或购买理由的核心表达弱项，有数据或强对照支持。
- P2：可优化但不是当前主要阻断的细节。

没有经营数据时，把 P0/P1/P2 改成「上线前风险顺序」。

## 单变量验证

每轮只改一个主要变量：

```text
假设：____
改动变量：____
保持不变：价格 / 标题 / SKU / 评价 / 流量来源 / 投放设置 / 统计时间
观察指标：____
最低样本量：____
继续条件：____
停止条件：____
下一轮：____
```

如果必须同时改多个变量，明确写「本轮只能验证整体方案，不能拆分归因」。

## 输出归因时必须写

- 指标口径
- 样本量
- 统计时间
- 对照是否存在
- 当前结论强度
- 竞争假设
- 下一步验证动作
