# 商品页改版 brief

本文件用于用户要求「怎么改」「给我改版方案」「给设计/生图工具的 brief」「下一版主图/详情页怎么做」。

## 改版原则

1. 先改阻碍用户理解和决策的信息，不先追求风格高级。
2. 每张图只承担一个主要任务。
3. 改文案时保留事实边界，不夸大功效和结果。
4. 改视觉时说明主体、场景、层级、文字和不能出现的元素。
5. 每轮只改一个主要变量，方便验证。

## 主图 brief 模板

```markdown
## 主图 brief：{商品}

目标：验证____。
目标用户：____。
使用场景：____。
主视觉：____。
必须出现：____。
必须写清：____。
不要出现：____。
文字层级：
1. {最重要信息}
2. {辅助信息}
3. {可选信息}
合规限制：____。
保持不变：____。
验证指标：____。
```

## 详情页 brief 模板

```markdown
## 详情页 brief：{商品}

页面目标：提高____阶段的理解或信任。
首屏承接：____。
模块顺序：
1. {产品确认}
2. {购买理由}
3. {规格/SKU}
4. {证据}
5. {疑虑消除}
必须补的证据：____。
必须删或降级的表达：____。
保持不变：____。
验证指标：____。
```

## 修改动作写法

写具体动作，不写空泛建议。

弱：

- 优化排版
- 突出卖点
- 增强信任
- 做得更高级

强：

- 把产品主体放大到画面 60% 以上，删除右上角第二个促销贴纸。
- 主图只保留「2 Pack」和核心用途，颜色选项放到副图。
- 详情页第一屏先解释尺寸适配范围，再讲材质。
- 删除「100% effective」这类无法证明的功效表达，改成用户可观察结果。

## 生图或设计交接

给图像工具或设计师时，需要说明：

- 商品是什么，不要只写品类。
- 画面比例和使用位置。
- 主体、背景、场景、人物和道具。
- 文字内容、语言、大小和层级。
- 不能改动的事实，例如颜色、容量、数量、材质。
- 禁止出现的夸张效果、品牌元素、违规符号。

如果用户上传了原图，需要明确：

```text
保留：____
删除：____
新增：____
调整：____
不能改变：____
```

## 验证计划

改版方案必须配一个验证动作：

```text
本轮只验证：____
看哪个指标：____
统计多久：____
样本不足怎么办：____
达到什么条件继续：____
低于什么条件回滚或换方案：____
```

没有验证计划的改版，只是审美修改，不是业务诊断。
