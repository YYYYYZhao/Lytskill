# 商品主图与详情页诊断知识包

这个知识包用于诊断商品页面有没有把产品翻译成用户能快速理解、愿意点击、愿意加购和愿意下单的信息。

## 购买链路与诊断边界

商品页面处在这条链路中：

```text
曝光
→ 主图、标题、售价展示等共同影响点击
→ 详情页、价格、商品规格与 SKU 组合、评价等共同影响加购
→ 用户实付总价、优惠、运费、物流、信任等共同影响下单
```

主图和详情页都只是成交变量之一。

| 数据表现 | 页面可能负责什么 | 同时要排除什么 |
| --- | --- | --- |
| 有曝光、点击率低 | 主图是否让用户看懂并产生点击理由 | 标题、售价展示、销量评价、曝光人群 |
| 有点击、加购率低 | 详情页是否建立购买理由 | 商品需求、价格、SKU、评价和店铺信任 |
| 有加购、下单率低 | 风险与交易条件是否清楚 | 用户实付总价、运费、优惠、物流时效 |
| 有下单、退款率高 | 页面是否造成预期错配 | 质量、发错货、履约、使用门槛 |

图片取证回答「页面上展示了什么、哪些信息需要核对」。

数据归因回答「页面是否可能是当前主要卡点」。

两者不能混为一谈。

只有图片时，不直接判断最前面的阻断层，不给 P0/P1/P2 经营优先级，也不输出完整改版方案。先根据用户目标补充最小数据：

- 主图点击：曝光量、点击量或点击率、统计时间、售价、评价、销量、标题或商品卡展示，以及可用对照。
- 详情页加购或下单：访问量或点击量、加购量、下单量、指标口径、用户实付金额、商品规格与 SKU 组合、评价、运费、物流方式和退换货条件。
- 竞品页面：公开销量、售价和评价只能补充经营背景，不能代替曝光和点击数据。

如果商品尚未上线且没有历史数据，可以做上线前信息风险检查，只判断信息正确性、规格清晰度和误导风险，不把检查结果写成点击或下单归因。

## 主图诊断框架

主图先解决两个问题：

1. 用户能否快速看懂这是什么。
2. 与同屏商品相比，用户为什么愿意点这一张。

### 1. 一眼看懂

做 3 秒测试：

- 这是什么产品？
- 主要解决什么问题？
- 使用之后能得到什么变化？

第一个问题答不上来，是产品识别问题。

第一个能回答，后两个答不上来，是购买理由问题。

不是每张主图都必须同时说完三个答案。标准品可能只需要清晰识别和差异，功能型商品可能需要直接展示结果或场景。

### 2. 主体与清晰度

检查：

- 产品主体是否清晰、完整、无遮挡。
- 缩小到手机商品列表尺寸后，产品是否仍然可辨认。
- 背景、道具、模特和文字是否抢走主体。
- 颜色、材质、纹理和关键结构是否接近真实商品。
- 图片压缩、抠图边缘、光影和比例是否造成廉价或失真感。

清晰不是只看分辨率。高分辨率但主体太小、背景太乱，仍然无法有效传递信息。

### 3. 点击理由

主图至少要让用户感知一种理由：

| 角度 | 画面表达 |
| --- | --- |
| 痛点 | 展示用户正在经历的问题 |
| 结果 | 展示使用前后或完成后的变化 |
| 场景 | 展示具体使用时刻和环境 |
| 差异 | 展示与同类产品不同的结构、功能、组合或风格 |
| 选择 | 展示颜色、数量、套装或适用对象 |

「好用」「高级」「品质好」不属于具体购买理由，除非画面能证明它。

### 4. 信息效率

用户不会逐字阅读主图。

检查：

- 核心信息是否在第一视觉层级。
- 文字是否能在手机端读清。
- 一张图是否塞入多个互相竞争的卖点。
- 图标、参数和促销信息是否遮挡产品。
- 文案是否只是形容词，没有场景、结果或证据。
- 语言是否完成目标市场本地化。

主图不是把详情页压缩成一张图。它只需要完成当前最关键的点击任务。

### 5. 差异与竞品环境

单独看好看的主图，放进搜索结果可能毫无识别度。

对比时检查：

- 同屏竞品都在用什么背景、角度和卖点。
- 当前主图是否与竞品采集图完全相同。
- 差异是否与购买有关，而不只是换颜色或加装饰。
- 差异是否建立在相近商品、价格带和目标用户上。

不能因为竞品销量高，就断言它的主图更有效。销量还受到链接历史、评价、价格、广告和渠道影响。

### 6. 真实与风险

检查：

- 图片颜色是否可能与实物偏差过大。
- 尺寸比例是否容易误导。
- 套装数量、赠品和包装内容是否表达准确。
- 功效前后对比是否可能夸大。
- 模特、场景或合成图是否暗示产品具备不存在的功能。
- 品牌标识、认证和授权是否有依据。

让用户误解的主图可能提高点击，却会提高退款、差评和售后。

## 详情页诊断框架

详情页不是信息越多越好。

它的任务顺序是：

```text
理解产品
→ 建立购买理由
→ 提供证据
→ 消除疑虑
→ 帮助选择
→ 说明交易与使用边界
```

### 1. 首屏定位

先用一句话复原：

```text
这个商品帮助「谁」在「什么场景」更快获得「什么结果」。
```

如果详情页首屏无法支持这句话，检查：

- 只展示产品，没有说明场景或结果。
- 同时对多个完全不同的人群说话。
- 卖点过多，没有第一优先级。
- 首屏使用空洞形容词，用户不知道具体改变是什么。

### 2. 产品理解

用户至少要看懂：

- 产品是什么。
- 包含哪些物品。
- 如何使用、安装或穿戴。
- 关键功能和限制是什么。
- 尺寸、材质、颜色、容量、数量或兼容范围是什么。

按商品需要选择，不要求所有页面机械罗列全部字段。

### 3. 购买理由

详情页需要回答「为什么买」：

- 当前问题是什么。
- 产品怎样解决。
- 使用后的可见结果是什么。
- 为什么选择这一款，而不是替代方案或竞品。

常用表达：

- 痛点：用户现在遇到什么麻烦。
- 结果：产品带来什么具体变化。
- 场景：什么时候最需要它。
- 对比：旧方案和新方案有什么差异。
- 结构：某个设计为什么能产生结果。

购买理由必须与商品真实能力一致。

### 4. 证据与信任

检查有没有与承诺匹配的证据：

| 承诺 | 更匹配的证据 |
| --- | --- |
| 材质更好 | 材质近景、成分或检测信息 |
| 尺寸合适 | 尺寸图、参照物、模特信息 |
| 效果明显 | 使用过程、前后对比、限制说明 |
| 结实耐用 | 结构细节、测试过程、使用条件 |
| 容易使用 | 步骤图、安装图、常见错误 |
| 值得信任 | 买家实拍、评价、授权或认证 |

评价数量、销量和店铺评分通常由平台组件呈现。截图里看不到时，不假设它们存在或不存在。

### 5. 风险与疑虑

商品详情页要主动回答用户可能不买的理由：

- 尺寸会不会不合适。
- 颜色是否有偏差。
- 材质是否安全、柔软、耐用或容易清洁。
- 是否兼容用户已有设备。
- 是否需要安装，安装是否困难。
- 套装包含什么，不包含什么。
- 发货、配送和退换货条件是什么。
- 功效和适用范围有哪些边界。

用户看完仍需猜测的关键问题，就是详情页风险。

### 6. 商品规格与 SKU 组合

商品规格与 SKU 组合包括：

- 颜色
- 尺寸
- 数量
- 单件或套装
- 不同 SKU 对应的售价

检查：

- SKU 名称能否与图片一一对应。
- 用户是否知道自己会收到什么。
- 最低展示价是否来自少量或无关 SKU，造成价格落差。
- 套装、赠品和数量是否容易误选。
- 详情页承诺与实际可选 SKU 是否一致。

### 7. 信息顺序与重复

推荐顺序不是固定模板，但必须符合用户判断过程：

1. 先告诉用户这是什么、适合谁。
2. 再给核心购买理由。
3. 再用细节、场景和证据证明。
4. 最后处理尺寸、使用、物流、售后和限制。

常见问题：

- 多张图重复同一个形容词。
- 先讲品牌故事，后讲产品是什么。
- 关键尺寸和限制藏在最后。
- 卖点、参数、使用方法和售后混在一张图。
- 长图过长，却没有新的信息增量。

### 8. 手机端扫读

检查：

- 文字是否过小。
- 一张图是否出现过多段落。
- 标题、结论和证据是否有视觉层级。
- 图片是否被平台压缩后仍可辨认。
- 本地化文字是否自然，是否出现未翻译中文。
- 用户扫视几眼能否抓到关键购买理由。

## 不同商品的表达差异

不要用同一套页面模板诊断所有商品。

### 功能型商品

优先检查：

- 问题是否具体。
- 功能如何产生结果。
- 使用步骤与限制。
- 参数、材质和效果证据。

### 情感与风格型商品

优先检查：

- 目标人群与审美是否清楚。
- 具体穿搭、送礼、家居或身份场景。
- 颜色、质感、细节和真实上身效果。
- 风格表达是否与目标市场一致。

### 标准化商品

优先检查：

- 型号、兼容性、规格和价格信息。
- 与同类商品的明确差异。
- 是否减少选错的风险。

### 非标准化商品

优先检查：

- 款式、做工、材质、尺寸和真实细节。
- 不同颜色和 SKU 的实际效果。
- 价格与品质感是否匹配。

### 高决策成本商品

需要更多证据、对比、参数、信任和售后信息。

### 低决策成本商品

优先提高信息效率，快速说清产品、场景、结果和交易条件，避免把简单购买写成一篇说明书。

## 对比诊断

### 多个主图版本

统一比较：

- 产品识别
- 主体清晰度
- 点击理由
- 差异性
- 真实可信
- 手机端可读性
- 与目标人群和场景的匹配

不要把「设计更复杂」或「看起来更高级」直接等同于更有效。

### 当前页面与竞品

只比较：

- 用户首先看到什么。
- 竞品主打什么购买理由。
- 对方如何证明。
- 对方消除了哪些疑虑。
- 自己缺什么或重复什么。

竞品适合作为表达参照，不是因果证明。

### 修改前后

需要记录：

- 改了哪张图。
- 改了什么元素。
- 同期是否修改标题、售价、优惠、SKU、评价或投放人群。
- 修改前后的曝光、点击、加购和下单样本。

多个变量一起改，只能判断整套方案变化，不能归因到某一张图。

## Top 店铺主图共性归纳

这个模式用于用户批量提供 Top 店铺、高销量商品或竞品主图，并要求找到共性、提炼主图打法或沉淀到 skill。

它回答的是：

```text
这些样本反复使用了哪些主图表达结构？
这些结构可以转成哪些可测试的改图方案？
```

它不回答：

```text
这些商品为什么卖得好？
这些主图是否直接导致高点击率或高成交？
```

销量、Top 店铺身份和公开销售结果只能证明「这些样本值得作为表达参照」，不能证明主图是销量原因。

### 1. 样本边界

先记录：

- 统计时间。
- 平台、国家和类目路径。
- 样本数量。
- 店铺名、商品名或主图可见品牌。
- 销量或用户用于判断 Top 的依据。
- 图片文件、链接或截图来源。

图片路径包含国家与类目时，优先从路径提取；用户后续修改路径或类目时，以最新路径为准。

跨子类目样本必须分组。不要把口红、香水、香皂、沐浴护理强行混成同一个主图公式。

### 2. 统一标签维度

每张主图至少标注这些字段：

| 维度 | 记录什么 |
| --- | --- |
| 主图类型 | 纯产品、产品加包装、模特使用、场景图、信息图、促销图、套装图 |
| 第一视觉 | 用户第一眼更可能看到产品、模特、价格、功效、品牌、赠品还是色号 |
| 产品呈现 | 主体占比、是否完整、是否有外盒、是否展示容量/规格/色号 |
| 购买理由 | 色号、质地、香调、功效、成分、套装、赠品、价格利益、官方店 |
| 信任元素 | 包装、官方店、认证/批准号、No.1、模特、品牌历史、销量/评价露出 |
| 本地化元素 | Malay 文案、RM、穆斯林模特、本地成分词、本地使用场景 |
| 信息密度 | 无文字、少量标题、多卖点、参数表、促销卡片 |
| 手机端风险 | 文字过小、产品太小、信息过密、主体被装饰抢走、裁切风险 |
| 合规风险 | 美白、医疗功效、认证、授权、inspired by 大牌、夸大前后对比 |

### 3. 共性提炼方法

先按子类目找共性，再看跨类目共性。

可写为共性的内容：

- 半数以上样本反复出现的画面结构。
- 同一子类目中反复出现的购买理由。
- 多个高销量样本同时使用的信任或本地化元素。
- 与用户当前主图相比明显缺失的表达模块。

不可写为共性的内容：

- 只有一张图出现的个别元素，除非它是用户特别要求研究的方向。
- 只因为设计更高级、颜色更统一，就判断它更有效。
- 只因为销量高，就判断该元素应该照抄。

### 4. 马来西亚美妆样本提示

分析马来西亚美妆个护主图时，特别关注：

- 口红与唇彩：色号、质地、妆效、包装、是否有色号矩阵、真人上唇或本地模特。
- 香水：容量、香调、灵感来源、味道结构、包装质感。
- 香皂和沐浴护理：功效、成分、套装内容、KKM 或批准号、本地护理场景。
- 本地化信号：Malay 文案、RM、穆斯林女性模特、Madu/Kelulut/Bidara 等本地常见成分词。
- 交易信号：Official Store、Free Gift、Net Spend、促销价、套装数量。

这些只是观察维度，不是固定答案。没有在样本中出现时不要编造。

### 5. 转成可测试方案

把共性转成「主图模板」，不要转成绝对规则。

示例：

```text
模板：产品加包装加核心功效。
适用：口红、唇膏、功效型小件美妆。
画面：产品主体占第一视觉，外盒在侧边证明正品和款式。
信息：品名、质地/功效、色号或容量。
假设：用户能更快确认收到什么，并理解购买理由。
验证：只替换主图，保持标题、价格和投放条件不变，观察点击率。
```

常见模板：

| 模板 | 适用场景 | 核心任务 |
| --- | --- | --- |
| 正品包装型 | 品牌化口红、香水、小件美妆 | 证明款式、正品感和质感 |
| 功效信任型 | 皂类、身体护理、功效型护肤 | 快速说明功效、成分和认证 |
| 本地模特场景型 | 口红、沐浴套装、香氛护理 | 建立目标人群和使用代入 |
| 选择促销型 | 多色号、多 SKU、活动款 | 说明色号、赠品、门槛和价格利益 |

### 6. 输出护栏

Top 样本归纳输出必须包含：

- 样本边界。
- 样本标签表。
- 高频共性。
- 子类目差异。
- 可测试优化方案。
- 不能推断的部分。

不要输出：

- 「因为这些主图这样做，所以销量高」。
- 「这个元素一定提升点击率」。
- 跨子类目的硬套模板。
- 未经核实的认证、授权、功效或平台规则。

## 数据归因与样本护栏

### 指标口径

```text
点击率 = 点击量 ÷ 曝光量
加购率 = 加购量 ÷ 点击量
点击到下单转化率 = 下单量 ÷ 点击量
```

如果平台使用买家数、订单数或成交件数作为分子，要保留平台原口径。

### 样本护栏

- 曝光少于 500：只看初始信号，不判断稳定点击表现。
- 点击少于 50：不能因为 0 下单断言详情页、价格或人群有问题。
- 预期订单小于 1 时，0 下单通常可能属于正常波动。
- 优先使用用户历史、店内同批商品、修改前后版本或同类竞品作对照。
- 固定点击率、加购率和转化率只能作为用户当前业务的参考区间，不能跨类目当硬标准。

### 归因强度

| 证据 | 能下的结论 |
| --- | --- |
| 只有一张图片 | 图片中存在某项可见信息或待核对风险，不能判断经营优先级 |
| 图片加低样本数据 | 问题与数据表现方向一致，待补样本 |
| 修改前后只改一个变量，样本足够 | 该改动可能影响指标 |
| 多轮重复且有对照 | 对当前商品和流量条件有较强支持 |

不要因为改图后指标上涨，就排除流量人群、活动、季节和竞争变化。

## 改版与验证

### 优先级

| 级别 | 定义 |
| --- | --- |
| P0 | 用户看不懂产品，或关键信息存在错误、误导和重大风险 |
| P1 | 产品能看懂，但购买理由、证据或关键疑虑不足 |
| P2 | 页面已经成立，属于信息顺序、重复、风格和效率优化 |

### 修改动作必须具体

不要写：

- 优化主图
- 丰富详情页
- 增强吸引力
- 提升高级感

要写：

- 把产品主体从画面宽度的约 30% 放大到约 60%，删除右侧无关道具。
- 第一张副图改成使用前后对比，保留同一角度和光线。
- 把尺寸信息从详情页末尾提前到 SKU 说明之后，并增加参照物。
- 删除三张重复的「高品质」文案图，替换为材质近景、使用步骤和包装清单。

比例和数量按实际图片调整，不机械套用。

### 单变量测试

```text
当前对象：商品 A 主图。
当前问题：曝光足够但点击率低于店内同批商品。
原因假设：产品主体过小，核心差异无法在列表页识别。
修改动作：只替换主图，标题、售价、优惠和投放条件保持不变。
验证指标：点击率。
对照：修改前 7 天或同等曝光样本。
下一步：指标改善则保留；无改善则检查标题、售价展示和曝光人群。
```

详情页测试同理，优先观察加购率和点击到下单转化率。

## 常见误判

### 误判 1：白底图一定不好

白底图可能清晰、直接，也可能与竞品同质化。要看品类、搜索环境、主体识别和点击理由，不能只看背景形式。

### 误判 2：文字越多越专业

用户通常只扫视几眼。文字多但没有信息层级，会降低理解效率。

### 误判 3：场景图一定比产品图好

场景图能建立代入感，但可能让产品变小、结构不清。主图是否使用场景，要看当前最重要的是识别、结果还是风格。

### 误判 4：详情页越长越完整

长度不等于完整。重复信息、空洞形容词和无关品牌故事不会增加购买理由。

### 误判 5：点击率低就是主图问题

主图重要，但标题、售价、销量评价、曝光人群和竞品环境也会影响点击。

### 误判 6：有点击没下单就是详情页问题

还要检查需求、商品本身、用户实付总价、SKU、评价、运费、物流和信任。

### 误判 7：竞品卖得好，所以照抄页面

竞品销量还来自链接历史、价格、评价、广告、达人和供应链。页面只能作为表达参照。

### 误判 8：一次重做全部页面

全部重做可能改变结果，但无法知道哪个动作有效。先修最前面的阻断层，再做单变量验证。
