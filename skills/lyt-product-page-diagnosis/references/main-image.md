# 主图与商品卡诊断

本文件用于主图、商品卡、列表页点击理由和多套主图方案对比。

## 主图承担的任务

主图不是海报，核心任务是：

1. 一眼看出卖的是什么。
2. 一眼看出和用户有什么关系。
3. 给出点击进入详情页的理由。
4. 在同屏竞品里不过度吃亏。
5. 不制造规格、数量、功效或品质误解。

## 五层诊断

### 1. 产品识别

检查：

- 主体是否清楚，是否被文字、贴纸、道具或背景淹没。
- 产品形态、颜色、尺寸、数量、套装关系是否可判断。
- 主图是否同时放太多 SKU，导致用户不知道实际卖什么。
- 缩小到手机列表页尺寸后，核心信息是否还可读。

常见问题：

- 卖的是套装，但图像像单件。
- 多颜色展示像随机发货。
- 产品主体太小，场景比商品更抢眼。
- 主图只表达氛围，不表达商品。

### 2. 使用场景

检查：

- 场景是否让用户知道什么时候用、谁来用、解决什么问题。
- 场景是否与目标国家、季节、文化和生活方式冲突。
- 模特、肤色、年龄、空间、道具是否制造不必要距离感。
- 场景是否真实可信，还是像图库拼贴。

没有场景不一定错；标品、低价消耗品和强规格商品可能更需要清晰产品图。

### 3. 点击理由

主图可用的点击理由包括：

- 明确功能：解决一个具体问题。
- 明确差异：比常见款多一个可见优势。
- 明确结果：展示使用前后或完成状态。
- 明确套装：数量、组合、赠品、适用对象。
- 明确价格感：大包装、优惠、价值感。
- 明确信任：认证、评价、真实使用，但必须有依据。

点击理由不足时，不要直接说「不吸引人」，要指出用户为什么没有必要点进去。

### 4. 信息效率

检查：

- 文字是否过多，是否挤压商品主体。
- 文案是否能被目标市场用户理解。
- 卖点是否集中，不要同时讲 5 个理由。
- 促销、功效、规格、认证是否抢同一视觉层级。
- 是否有拼写错误、单位错误、语言混杂或语义不清。

主图文字建议只保留「影响点击的一句话」，更多解释放到副图或详情页。

### 5. 真实与风险

检查：

- 效果图是否夸张到可能引发差评或退款。
- 功效、认证、材质、容量、尺寸是否有证据支持。
- 前后对比是否可能被平台判定夸大。
- 是否存在品牌、人物、IP、医疗、美妆功效等合规风险。

风险不一定代表不能做，但必须从「卖点」降级为「待确认」。

## 对比诊断

对比多张主图时，先统一：

- 同一个商品或同一子类目
- 同一国家/平台
- 同一价格区间
- 同一展示尺寸
- 同一目标：点击、解释规格、建立信任或减少误解

输出不要说「A 比 B 好看」，而要说：

```text
A 更强在____，适合验证____。
B 更强在____，适合验证____。
当前不能判断点击效率，因为还缺____。
```

## 主图改版方向

常见改法：

- 放大主体，减少非必要装饰。
- 把多 SKU 拆成「主推款 + 可选颜色」。
- 用一张图只表达一个点击理由。
- 把强规格信息前置，例如数量、尺寸、容量、适用型号。
- 用真实使用场景替代泛化氛围图。
- 把功效或认证改成可证明、可合规的表述。

每次改版只测一个主要变量，否则后续无法归因。
