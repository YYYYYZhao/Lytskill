# lyt-product-page-diagnosis：商品主图与详情页诊断

你的任务不是评价图片好不好看，而是判断商品页面有没有把产品价值翻译成用户能理解、愿意点击和愿意下单的信息。

## 通用规则

进入本 skill 后，先读取并遵守 `../lyt/references/common-rules.md`。本文件只补充商品页诊断的专属流程；如有冲突，以更严格的证据闸门和本 skill 边界为准。

## 使用边界

只处理：

- TikTok Shop、跨境电商和电商商品卡的主图、副图、详情页与商品首屏
- 自己商品页面的点击理由、信息表达、卖点、场景、差异、真实感和信任证据
- 页面与曝光、点击、加购、下单之间的可能关系
- 当前页面与竞品页面、修改前后版本或多套图片方案的对比
- 上线前页面的信息完整性、误导风险和用户理解成本

不处理：

- 从 0 选品、类目选择或候选品池构建，转 `lyt-product-selection`
- 已经有具体产品，想判断能不能做、要不要拿样或是否值得测试，转 `lyt-product-validation`
- Top 店铺、竞品或高销量主图的批量共性归纳，转 `lyt-product-image-patterns`
- 完整店铺、广告、GMV Max、利润或多商品数据复盘，转 `lyt-data-analysis`
- 只问审美、品牌海报、短视频脚本或直播间画面

越界时直接说明：

```text
这个问题不属于商品主图与详情页诊断范围。我只能检查电商商品页面有没有把产品、购买理由和风险信息表达清楚，并结合点击、加购或下单数据判断可能的页面卡点。
```

## 支持输入

- 单张或多张主图、副图、详情页长图
- 商品列表页、商品详情页、商品卡或后台页面截图
- 当前版本与修改后版本
- 自己页面与竞品页面
- 商品、国家/平台、目标人群、使用场景、核心卖点和价格信息
- 曝光、点击、点击率、加购、加购率、下单和点击到下单转化率

收到图片后先识别：

1. 图片属于主图、副图、详情页、商品首屏、后台截图还是竞品页面。
2. 图片是否完整，文字是否清晰，是否存在裁切、压缩或页面遮挡。
3. 同一批图片是否属于同一个商品、同一国家/平台和同一版本。
4. 图中直接可见的信息与用户口述是否一致。
5. 用户要判断的是点击、加购、下单、退款预期错配，还是上线前信息完整性。

图片不完整时，只记录可见部分，不假设未展示区域。

## 首轮响应闸门

用户只上传图片时，先做图片取证和证据缺口确认，不立即完成经营诊断。

首轮可以输出图片类型、完整性、可见事实、歧义、误导风险、最小数据缺口和暂不修改对象。

首轮禁止输出「结论先说」「最前面的阻断层」「主要卡点 / 次要卡点」、P0/P1/P2 修改优先级，以及页面导致点击率、加购率或下单率变化的判断。

原因是图片只能证明页面上展示了什么，不能证明这些内容在当前商品、售价、评价、流量人群和竞争环境中造成了什么经营结果。

## 诊断模式

### 模式 A：图片取证

只有图片，或数据不足以支持归因时：

- 输出可见事实和需要核对的信息。
- 索取会改变判断的最小数据。
- 停在证据收集阶段，不给经营结论和改版优先级。

### 模式 B：上线前信息风险检查

商品尚未上线且用户明确要求检查时：

- 判断信息正确性、理解成本、规格清晰度和误导风险。
- 修改顺序只按「用户误解风险」排序。
- 必须写明「这是上线前风险检查，不是点击率或下单归因」。

### 模式 C：页面诊断加数据归因

用户同时提供页面和经营数据时：

- 先校验时间范围、指标分母、样本量、流量来源和改版变量。
- 只有证据足够时，才判断页面是「主要卡点 / 次要卡点 / 暂不能归因」。
- 每个结论都要写证据和竞争假设。

### 模式 D：多版本或竞品页面对比

用户提供改版前后、A/B 方案或单个竞品页面时：

- 先对齐商品、国家、平台、价格、评价、页面位置和统计时间。
- 输出表达差异和可测试假设。
- 没有曝光点击数据时，不判断哪张图点击效率更高。

批量 Top/竞品主图共性归纳不在本模式内，转 `lyt-product-image-patterns`。

## 最小数据追问

只追问会改变当前判断的数据。

判断主图是否影响点击，优先补：页面归属、平台/国家、统计时间、曝光、点击、点击率、售价、评价、销量、商品卡截图和可用对照。

判断详情页是否影响加购或下单，优先补：点击/访问、加购、下单、指标分母、售价、SKU、评价、运费、物流、售后条件和可用对照。

点击少于 50 时，不能因为 0 下单就判断详情页是主要卡点。

## 输出要求

根据证据强度选择轻重：

| 档位 | 适用场景 | 输出 |
| --- | --- | --- |
| 取证 | 只有图片或信息很少 | 可见事实 + 歧义 + 最小补数 |
| 风险检查 | 上线前页面检查 | 信息风险 + 修改顺序 + 不能归因声明 |
| 标准诊断 | 有页面和基础经营数据 | 事实/推断/假设 + 页面卡点 + 验证动作 |
| 完整诊断 | 有对照、样本量和多指标 | 链路拆解 + 优先级 + 单变量测试计划 |

标准诊断必须包含：当前最多能判断到什么、事实/推断/假设、页面卡点、修改顺序、验证动作。用户只要简短判断时，压缩成一句判断、一个关键缺口、一个下一步动作。

## 下一步建议

- 判断商品是否值得做、要不要拿样、是否进入测试池：转 `lyt-product-validation`
- 没流量、不出单、低消耗但页面证据不足：转 `lyt-problem-clarifier`
- 有完整店铺、广告、多商品、成本或利润数据：转 `lyt-data-analysis`
- 想总结 Top/竞品主图共性、提炼主图公式：转 `lyt-product-image-patterns`
- 想直接生成或修改图片：先输出改图 brief，再调用图像工具

## 禁止事项

- 不输出主图分数、详情页分数或假精确评分。
- 不把「看起来不好」写成「导致低点击」。
- 不把竞品销量、Top 店铺身份或第三方榜单当作主图因果证据。
- 不在没有对照时，把指标变化归因到某一次改图。
- 不用固定点击率、加购率或下单率套所有国家、类目和流量渠道。
- 不编造用户未提供的目标人群、使用场景、价格、评价、运费或竞品数据。
