# 学习状态与反馈规则

本文件用于 `lyt-learning` 的对话学习、反馈提取、知识检验交接和学习状态卡。

## 反馈提取

继续下一篇时必须：

1. 读取当前对话里的上一篇内容和用户反馈。
2. 忽略默认反馈提示问题，只提取用户真实填写的内容。
3. 总结用户当前理解状态。
4. 再生成下一篇。

如果用户没有反馈，先问：

```text
你读完后先告诉我三件事：
1. 哪里看懂了？
2. 哪里没看懂？
3. 哪个地方想展开？
```

如果用户明确说「直接继续」，可以小步推进，并说明本篇基于当前可见上下文生成。

## 反馈分类

| 反馈类型 | 处理 |
| --- | --- |
| 没看懂 | 降低难度，增加例子 |
| 看懂但没兴趣 | 换角度，连接真实问题 |
| 想应用 | 增加案例和判断方法 |
| 已掌握 | 提高概念密度 |
| 提出具体问题 | 先回答问题，再继续课程 |
| 反馈很少 | 保持当前难度，小步推进 |
| 事实纠错 | 判断是否有新事实、可靠来源或逻辑错误 |

反馈可以调整讲解角度和节奏，但不能自动改变事实结论。

## 反驳处理

用户反驳文章观点时，先区分：

- 学习反馈
- 不同意见
- 替代解释
- 新事实
- 可靠来源
- 逻辑纠错

只有新事实、可靠来源或逻辑纠错，才修改知识结论。结论变化只允许「维持 / 收窄 / 推翻」。

## 学习状态卡

对话模式只在回答末尾更新学习状态卡，不写文件。

学习状态统一记录为：

```text
学习目标：____
已完成：____
已掌握：____
当前缺口：____
下一步：学习 / 知识检验
```

状态卡只写已经发生的学习和反馈，不编造掌握程度。

## 知识检验交接

转入 `lyt-knowledge-check` 时，带上：

- 学习主题
- 最新文章主题
- 用户反馈
- 已掌握内容
- 最大待检验缺口
- 需要检验的核心能力：概念 / 证据 / 因果 / 边界 / 验证

知识检验返回后，下一篇只补最大缺口，不重新讲完整课题。
