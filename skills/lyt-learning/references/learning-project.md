# 文件化学习工程

本文件只在用户明确要求保存课程、生成本地文件、创建学习目录或继续本地课程时读取。

## 默认位置

用户没有指定目录时，把学习课题放在当前项目的：

```text
学习课题/{课题名}/
```

用户指定目录时，使用用户指定目录。

## 目录结构

每个课题使用固定结构：

```text
{课题名}/
├── 00-学习计划.md
├── 01.md
├── 02.md
├── 03.md
└── assets/
```

不要创建额外 README、CHANGELOG 或安装说明。

## 创建第一篇

如果用户给了课题，确认学习目标。只有用户明确要求文件化时，才创建本地课题目录、`00-学习计划.md` 和第一篇文章。

第一篇文章结构：

```markdown
# 01｜{标题}

## 这一篇要解决的问题

## 正文

## 小结

## 下一篇预告

---

## 学习反馈

你可以写：

1. 哪里看懂了？
2. 哪里没看懂？
3. 哪个地方想展开？
4. 这个主题和你的真实问题有什么关系？
5. 用你自己的话说，这一篇最核心的因果链是什么？

请写在这行下面：
```

## 继续下一篇

文件化模式继续时必须：

1. 读取 `00-学习计划.md`。
2. 找到最新一篇文章。
3. 读取文章末尾的「学习反馈」。
4. 忽略反馈区里的默认提示问题，只提取用户真实填写的内容。
5. 总结用户当前理解状态。
6. 再生成下一篇。

## 更新学习计划

每次生成文章后，更新 `00-学习计划.md`：

- 当前进度
- 本篇主题
- 用户上一轮反馈摘要
- 已掌握内容和判断依据
- 尚未掌握或待检验内容
- 下一篇方向
- 最近更新时间

## 完成提示

文件化模式完成一次生成后，告诉用户：

```text
已经生成：

- 学习计划：{路径}
- 本篇文章：{路径}

下一步：读完后，在文章末尾的「学习反馈」里写下你的问题、感悟或想展开的方向。下次说「继续下一篇」，我会先读反馈再写。
```
