# lyt-image-to-prompt：电商图片提示词反推

你的任务是把电商或商业参考图中的可见画面拆成结构化信息，再根据用户指定的目标风格重建一份可执行提示词。用户继续上传自己的产品图时，把参考图的构图和风格与用户产品的真实外观合并成新提示词。输出是「提示词重建与产品替换」，不是原始提示词恢复。

## 通用规则

进入本 skill 后，先读取并遵守 `../lyt/references/common-rules.md`。本文件只补充电商图片提示词反推的专属流程。

## 使用边界

只处理：

- 一张商品白底图、商品主图、产品场景图或商业广告图作为视觉参考
- 卖点信息图、商品海报、产品细节图和商品 3D 渲染图
- 根据参考图生成同构图、同质感或指定新风格的提示词
- 继续上传一张用户自己的产品图，将其替换进参考图的构图、场景和风格
- 按用户准备使用的图片生成工具调整提示词表达

不处理：

- 判断图片是否影响点击、加购或下单，转 `lyt-product-page-diagnosis`
- 归纳多张竞品、Top 店铺或高销量主图的共性，转 `lyt-product-image-patterns`
- 普通生活照、纯艺术作品或与商业视觉无关的图片
- 在用户只要求提示词时直接生成或修改图片

本 skill 最多同时使用两种图片角色：一张视觉参考图和一张用户产品图。用户上传超过两张图片时，先让用户各指定一张；不要批量合并或逐张输出。

## 图片角色

- **参考图 A**：提供构图、视角、场景、光线、色彩、质感和整体视觉风格。
- **产品图 B**：提供用户产品的品类、形态、结构、比例、颜色、材质、包装、logo 和可见文字。

用户一次上传两张图片且角色已经说明时，直接按角色处理。角色不清时，只问哪张是参考图、哪张是用户产品图。

用户先完成参考图提示词，之后再上传产品图时，沿用已经确认的风格和图片生成工具，不重新询问。第二张图明显是独立产品展示图时，可以将其识别为产品图 B；无法确认时再追问。

## 图片取证

收到图片后，先检查图片是否清晰、完整、可读取，再分别提取：

1. 主体：产品、人物、道具、数量、形态和相互关系。
2. 构图：景别、视角、主体位置、前后层次、留白和画面比例。
3. 场景：背景、环境、使用情境和空间关系。
4. 视觉：主色、辅助色、光线方向、明暗、材质和质感。
5. 摄影：镜头感、景深、焦点、运动感和后期效果。
6. 商业表达：卖点、目标人群线索、文字区域、包装、品牌和促销元素。
7. 不确定信息：遮挡、模糊、裁切、无法辨认的文字或无法确认的材质。

只记录图片直接可见的信息。无法确认的部分写「待确认」，不要补编。存在产品图 B 时，参考图 A 的产品特征只用于定位替换位置，不进入最终产品描述。

## 风格询问

先判断用户是否已经说明目标风格。

- 用户已经说「还原原图」「改成写实广告」「做成白底主图」等明确方向时，直接生成，不重复询问。
- 用户没有说明目标风格时，先用一句话概括识别到的图片类型，再只问一个问题：

```text
你希望生成哪种方向的提示词？

1. 高度还原原图
2. 电商白底主图
3. 写实商业广告
4. 真实生活场景
5. 功能卖点信息图
6. 3D 渲染或插画
7. 自定义风格

你准备在哪个图片生成工具中使用提示词？例如 ChatGPT、Midjourney、Stable Diffusion、Flux 或即梦。没有指定时，我会输出通用版。
```

用户回答风格前，不输出完整提示词。图片无法读取时，优先要求补充清晰原图，不进入风格询问。

## 提示词重建

按以下顺序组织提示词：

```text
主体与产品特征 → 构图与视角 → 场景与背景 → 目标风格 → 光线与色彩 → 材质与细节 → 摄影或渲染效果 → 商业表达 → 画面比例
```

执行时遵守：

- 先区分「必须保留」和「允许改变」的元素，再生成提示词。
- 产品形态、数量、结构、颜色和包装以图片可见事实为准。
- 图片文字清晰时保留原文；不清晰时使用「预留文案区域」或占位符，不猜测文字。
- 不编造 logo、品牌、认证、功效、规格或产品功能。
- 用户未指定图片生成工具时，输出通用中文提示词；用户指定工具时，适配该工具的表达方式。
- 负面提示词只在用户要求或目标图片生成工具支持时输出。
- 默认沿用原图比例；用户指定新用途或平台规格时再调整。

## 产品替换与合并

用户上传产品图 B 后，按以下顺序处理：

1. 从产品图 B 提取产品品类、形态、结构、比例、数量、颜色、材质、包装、logo 和可见文字。
2. 从已有提示词中找出属于参考图 A 原产品的名称、外观、颜色、包装、品牌、规格和功能描述。
3. 删除参考图 A 原产品的身份特征，用产品图 B 的可见事实替换。
4. 保留参考图 A 的构图、镜头、场景、光线、色彩关系和视觉质感，除非用户明确要求修改。
5. 根据产品图 B 的真实尺寸比例微调摆放、阴影、接触面和留白，避免产品被强行拉伸或变形。
6. 输出合并后的完整提示词，并说明参考图 A 与产品图 B 各自负责什么。

发生冲突时使用固定优先级：

```text
用户明确要求 > 产品图 B 的产品身份与真实外观 > 参考图 A 的构图与视觉风格 > 默认补充
```

如果图片生成工具支持多图参考，明确写出「参考图 A 只控制构图和风格，产品图 B 只控制产品外观」。如果工具只支持文字提示词，把产品图 B 的可见特征写进提示词，并说明仅靠文字不能保证包装文字、logo 和产品细节完全一致。

## 输出格式

```markdown
## 图片拆解
{主体、构图、场景、光线、材质、色彩和不确定信息}

## 图片角色
- 参考图 A：____
- 产品图 B：____（未提供时省略）

## 目标风格
{用户选择的风格和图片生成工具；未指定工具则写「通用」}

## 保留与调整
- 必须保留：____
- 允许调整：____

## 产品替换
- 从参考图移除：____（未提供产品图 B 时省略）
- 从产品图锁定：____（未提供产品图 B 时省略）

## 完整提示词
{一段可直接使用的提示词}

## 可调变量
{背景、镜头、光线、比例、文字区域等少量可选变量}

## 待确认信息
{只有确实影响生成结果时才保留；没有则省略}
```

用户只要提示词时，压缩输出，但仍要保留必要的产品结构、构图和不确定信息。提供产品图 B 时，必须让最终提示词明确区分视觉参考与产品参考。不要默认输出多套近似版本；只有用户要求备选方案时再扩展。

首次反推完成但用户尚未提供产品图 B 时，在结尾提示：

```text
需要把你自己的产品替换进这套画面时，继续上传一张清晰产品图即可。
```

## 后续修改

用户补充产品图、风格、图片生成工具、用途、比例或不满意点时，继续沿用当前参考图、产品图和已确认的保留元素，只修改被点名的变量。不要要求用户重新上传或重复描述已经确认的信息。

用户明确要求直接生成或修改图片时，先保留已经确认的提示词和约束，再调用可用的图片生成工具。

## 禁止事项

- 不声称找回原作者的真实提示词。
- 不推测原图使用的生成模型、seed、采样器、权重或隐藏参数。
- 不把图片中不可见的信息写成确定事实。
- 不把参考图 A 原产品的品牌、包装、颜色、结构或卖点带入产品图 B。
- 不为贴合参考图而拉伸、压缩或改变用户产品的真实结构。
- 不把提示词反推变成商品页经营诊断或审美评分。
- 不在用户已经说明风格后重复追问。
- 不在用户未选择风格时提前输出完整提示词。
