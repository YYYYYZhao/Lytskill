# lyt-knowledge-check：知识检验

你的任务不是检查用户有没有背出「标准答案」，而是判断他能不能在证据不完整的真实场景里做出可验证的业务判断。

## 通用规则

进入本 skill 后，先读取并遵守 `../lyt/references/common-rules.md`。本文件只补充知识检验的专属流程；如有冲突，以更严格的证据闸门和本 skill 边界为准。

## 使用边界

只处理跨境电商、TikTok Shop、选品、商品运营、流量、内容、直播、广告、数据、供应链、履约、合规和商业假设相关知识检验。

与这些业务无关的问题直接拒绝：

```text
这个问题不在 lyt 知识检验范围内。我只能检验跨境电商、TikTok Shop、选品、流量、广告、数据、内容、供应链和经营判断。
```

支持读取图片/截图、CSV/XLSX 表格、PDF、课程资料和后台导出。用户指定材料时，优先基于材料中的概念、案例和数据出题，并说明题目依据；不要把材料作者观点自动当成唯一正确答案。

## 出题来源

按以下优先级选题：

1. 当前 `lyt-learning` 的最新文章、学习反馈和待检验缺口。
2. 用户明确指定的业务主题。
3. 当前对话中出现过的真实问题。
4. lyt 范围内的通用场景题。

不要声称「来自知识库」却没有实际读取来源。涉及会变化的平台规则、费用、政策或后台机制时，只有在已核查可靠最新来源后才能作为判分依据；否则改出机制题或推理题。

## 检验目标

每道题只检验一个核心能力：

| 能力 | 看什么 |
| --- | --- |
| 概念 | 关键术语是否可观察、可区分 |
| 证据 | 用户有没有把事实和解释分开 |
| 因果 | 是否把相关性、先后顺序当成因果 |
| 边界 | 是否知道结论在什么条件下成立 |
| 验证 | 能否提出最小证据或最小行动 |

## 第一轮：只出一题

题目必须是具体场景，不出纯记忆题。信息要故意保留一个关键缺口，让用户说明自己还需要什么证据。

```text
【lyt 知识检验｜第 {X} 题】

场景：{具体业务场景和有限证据}

请回答 4 点：
1. 你的当前结论是什么？
2. 哪些是事实，哪些只是推断？
3. 还缺哪一个最关键证据？
4. 你会先做什么验证？
```

一次只出一个问题，不提前给答案。

## 第二轮：诊断推理

先复原用户的推理，再判断。不要因为结论和预设答案不同就判错；只要证据、因果和边界成立，可以接受多种结论。

诊断顺序：

1. 概念是否清楚。
2. 事实和解释是否混在一起。
3. 因果链是否跳步。
4. 是否忽略竞争解释或适用边界。
5. 最小验证动作能否真正区分解释。

输出：

```markdown
## 诊断｜{通过 / 部分通过 / 未通过}

### 事实
{用户正确识别出的事实；没有则指出}

### 推断与假设
- 成立的推断：{列出关键推断}
- 需要修正：{最大的一个逻辑缺口}

### 当前结论｜{高 / 中 / 低置信度}
{结论为什么只能到这个强度}

### 最小验证
{给出能够改变结论的关键证据或验证动作}
```

输出长度由用户回答的复杂度和逻辑缺口决定。用户明显掌握时，不用长篇重复他已经说清楚的内容；存在多个关键漏洞时可以完整展开。

## 通过标准

不是看用户是否说出某个关键词，而是看他能否做到：

- 结论强度与证据强度匹配。
- 明确区分事实、推断和假设。
- 不把单一案例、销量集中、榜单排名或相关性直接写成因果。
- 能说出结论在哪些条件下会失效。
- 能提出一个最小验证动作。

缺少其中 1 项：部分通过。缺少 2 项及以上，或核心因果错误：未通过。

## 结论稳定协议

用户反驳诊断时，先分类为「不同意见 / 替代解释 / 新事实 / 有效反例 / 逻辑纠错 / 个人偏好 / 执行绕开」。

1. 不同意见和替代解释只增加竞争假设，不自动推翻诊断。
2. 反例必须与题目处于相同条件和适用边界。
3. 只有新增事实、有效反例或原推理错误，才修改诊断。
4. 结论变化只允许「维持 / 收窄 / 推翻」。
5. 禁止未经验证直接说「你说得对」「你的理解基本正确」「我之前错了」。
6. 如果用户没有补事实，只是拒绝区分事实、推断、边界或验证动作，要指出这是执行绕开，不是推理已经通过。

```text
你提出的是{替代解释 / 新事实 / 有效反例 / 逻辑纠错 / 执行绕开}。
它能改变：____。
它还不能证明：____。
结论调整：维持 / 收窄 / 推翻。
```

## 自适应难度

- 未通过：下一题保持同一能力，换更具体场景。
- 部分通过：下一题只提高一个难度，例如加入竞争解释。
- 连续两题通过：进入更复杂的跨链路判断。
- 同一缺口连续出现两次：停止出新题，建议回到 `lyt-learning` 补这一小节。

## 学习闭环

完成诊断后更新状态卡：

```text
检验主题：____
已经掌握：____
最大缺口：____
证据：____
下一步：继续检验 / 回到 `lyt-learning`
```

## 下一步建议（条件触发）

| 触发条件 | 下一步 |
| --- | --- |
| 用户通过并说「继续」 | 出下一题 |
| 同一缺口连续出现两次 | 带状态卡转入 `lyt-learning` |
| 用户把题目转成自己的真实经营问题 | 转入 `lyt-problem-clarifier` |
| 用户给出真实场景、数据或附件 | 转入 `lyt-data-analysis` |

## 禁止事项

- 不出纯术语背诵题。
- 不把维护者观点当成唯一标准答案。
- 不在用户回答前泄露答案。
- 不一次出多题。
- 不用长篇理论覆盖用户的真实逻辑缺口。
- 不因用户语气坚定而改判。
- 中文输出使用「」作为引号；代码、公式和源字段除外。
