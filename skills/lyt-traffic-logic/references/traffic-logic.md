# 流量分配逻辑知识包

这个文件是 `lyt-traffic-logic` 的随包知识来源。它必须能随 skill 一起发布到 GitHub，不能依赖维护者本地的内容结构化系统路径。

使用时根据用户问题选择相关流量渠道；当多个流量渠道共同影响判断时，可以完整展开。

## 总判断

流量不是一个东西，而是不同流量渠道的分配结果。

```text
商品卡/搜索流量：用户已经有需求，核心是让用户搜到你、看到你、点你、相信你。
短视频/推荐流量：用户本来不一定想买，核心是内容能不能让用户停留、互动、继续看。
广告/GMV Max 流量：平台把有限广告位分给更可能带来当下收益和长期体验的商品、素材或广告，核心是 eCPM 排序。
```

解释流量问题时，先分清用户问的是哪种流量渠道：

| 流量渠道 | 用户状态 | 平台主要看什么 | 用户容易误解什么 |
| --- | --- | --- | --- |
| 商品卡/搜索 | 用户已经有明确需求 | 关键词、商品排名、点击率、转化反馈 | 以为上架后自然会被看见 |
| 短视频/推荐 | 用户原本不一定想买 | 完播、停留、互动、跳出、后续行为 | 以为掌握算法就能做爆款 |
| 广告/GMV Max | 平台要分配有限广告位 | 出价、预估点击率、预估转化率、用户体验 | 以为开广告就一定能花出去 |

## 商品卡/搜索流量

商品卡不是玄学概念，本质是搜索流量。搜索流量背后代表用户已经有明确需求。

| 环节 | 发生了什么 | 正确理解 |
| --- | --- | --- |
| 搜索关键词 | 用户主动搜索某个需求 | 搜索流量代表用户已经想买 |
| 商品排名 | 用户只能看到排名靠前的商品 | 新链接没权重，排名低，用户可能根本看不到 |
| 点击进入 | 用户被主图、标题、价格吸引 | 主图、标题、价格影响点击 |
| 详情判断 | 用户看评价、销量、物流、活动、详情页 | 下单前要消除疑虑 |
| 下单付款 | 购买理由大于不买理由 | 成交不是看见就发生，而是信任和理由成立 |

商品卡流量的一句话：

```text
商品卡流量不是「平台随机给」，而是用户先有搜索需求，平台再按商品排名和反馈把更值得展示的商品排到前面。
```

常见误区：

| 误区 | 正确理解 |
| --- | --- |
| 上架了就应该有曝光 | 用户搜索后也要能看到你，排名低就可能完全看不到 |
| 别人能出单，我也应该能出单 | 同一个关键词下，商品排名、链接权重、点击和转化反馈都不同 |
| 商品卡就是平台自动给流量 | 商品卡本质是搜索场景，要先有需求和关键词匹配 |

## 短视频/推荐流量

短视频推荐不是固定流量池数字，而是平台不断测试用户行为。

平台的目的不是帮商家卖货，而是让用户持续停留。用户刷得越久，平台能展示的内容和广告越多。

| 环节 | 平台在看什么 | 正确理解 |
| --- | --- | --- |
| 小范围测试 | 先推给一小批可能相关的人 | 平台也不知道内容好不好，所以先测试 |
| 用户行为 | 完播、停留、互动、跳出、退出 | 内容能不能让用户继续刷，是核心信号 |
| 继续推流 | 数据好就扩大，数据差就减少 | 推荐是持续赛马，不是一次性分配 |
| 爆款形成 | 每一轮反馈都好，才可能继续放大 | 算法是放大器，不是爆款制造机 |

短视频推流的一句话：

```text
短视频推流的核心不是掌握某个流量池数字，而是让内容产生能被平台继续放大的用户行为。
```

常见误区：

| 误区 | 正确理解 |
| --- | --- |
| 流量池有固定数字 | 这些数字多数只是观察总结，不是平台公开机制 |
| 理解算法就能做爆款 | 算法是放大器，不是爆款制造机 |
| 内容没流量就是账号问题 | 没证据前，优先理解用户行为反馈，而不是下账号判断 |

## 广告/GMV Max 流量

广告流量不是谁愿意花钱就给谁。平台卖的是有限广告位，所以要排序。

| 环节 | 平台在看什么 | 正确理解 |
| --- | --- | --- |
| 广告位有限 | 广告位数量来自页面浏览量和广告密度 | 不是每个广告都能获得足够曝光 |
| eCPM 排序 | 出价、预估点击率、预估转化率共同决定排名 | 出价只是其中一个变量 |
| 用户体验 | 互动、观看、投诉、跳出等行为会影响后续分配 | 平台收广告费，也要保护用户体验 |
| GMV Max | 围绕商品、素材、商品卡/搜索流量、预算和投入产出比目标找成交机会 | 低消耗不能只看投入产出比目标 |

必须讲清楚这个公式：

```text
eCPM = 出价 × 预估点击率 × 预估转化率 × 1000
```

广告流量的一句话：

```text
广告跑不跑得出去，不只看你愿不愿意花钱，还要看平台预估这次曝光能不能带来点击、成交和可接受的用户体验。
```

常见误区：

| 误区 | 正确理解 |
| --- | --- |
| 出价高就一定有流量 | 出价只是 eCPM 的一部分 |
| 开了 GMV Max 就一定花钱 | GMV Max 还要看商品、素材、商品卡/搜索流量和历史反馈 |
| 低消耗只需要降低投入产出比目标 | 低消耗还可能来自商品卡弱、素材缺失、链接无反馈 |

## 冷启动逻辑

新店、新品、新链接的问题，不只是没有流量，而是没有历史反馈。

```text
没人知道你 -> 没人点击/购买你 -> 没有历史反馈 -> 平台更没有理由继续给你流量
```

冷启动必须和 GMV 公式一起解释：

```text
GMV = 曝光 × 点击率 × 转化率 × 客单价
```

| 变量 | 新店/新链接的问题 | 单独解决为什么不够 |
| --- | --- | --- |
| 曝光 | 没人看到你 | 只有流量，没有点击和成交也没用 |
| 点击率 | 用户看到了但不点 | 只有曝光，点击弱，反馈还是差 |
| 转化率 | 用户点了但不下单 | 只有点击，没有购买理由也没用 |
| 客单价 | 有成交但价值低 | GMV 仍然起不来 |

冷启动的一句话：

```text
冷启动不是靠单一动作解决，而是要同时让曝光、点击和转化至少有一个基础反馈，平台才有继续测试的理由。
```

## 成交链路

有点击不代表会成交。点击只说明用户愿意看，成交要看购买理由有没有成立。

成交可以这样解释：

```text
成交 = 购买理由大于不买理由
```

| 链路 | 用户行为 | 影响因素 |
| --- | --- | --- |
| 有曝光没点击 | 看到了但没兴趣 | 主图、标题、价格、素材钩子 |
| 有点击没加购 | 愿意看但没被说服 | 详情页、价格、评价、购买理由 |
| 有加购没下单 | 有意向但仍犹豫 | 优惠、包邮、物流时效、信任 |
| 有下单没成交 | 临门一脚卡住 | 价格、优惠、物流、信任 |

成交链路的一句话：

```text
流量解决的是「看见你」，成交解决的是「为什么现在要买你」。
```

## 按问题选择讲解重心

| 用户问题 | 先讲什么 | 重点纠正的误区 |
| --- | --- | --- |
| 为什么店铺没流量 | 店铺流量来自商品卡、短视频、直播、广告、达人等多个流量渠道 | 不是开了店就会有流量 |
| 为什么商品没曝光 | 商品卡搜索先看需求、关键词、排名和链接权重 | 不是上架了就等于会被看到 |
| 为什么新店/新品没流量 | 冷启动是缺历史反馈导致的循环 | 新店扶持不等于直接大量曝光 |
| 为什么别人有流量我没有 | 平台会比较谁更相关、更容易被点、更容易成交、体验风险更低 | 流量不是平均分配，而是竞争分配 |
| 为什么短视频没流量 | 推荐流量看用户行为反馈 | 不是套一个流量池数字就能爆 |
| 为什么广告低曝光/低消耗 | 广告按 eCPM 和用户体验排序，GMV Max 还要看商品基础和可用素材 | 不是开广告就一定能花出去 |

## 输出组织原则

根据用户问题决定讲解范围，不设置固定长度。

建议按这个顺序组织：

1. 先说结论，指出底层逻辑。
2. 再分流量渠道：说明是商品卡/搜索、短视频/推荐，还是广告/GMV Max。
3. 用一张表讲清平台在看什么、用户误解什么、正确理解是什么。
4. 最后给一句用户能带走的话。
