# 主图样本归纳知识包

本文件用于从竞品、Top 店铺或高销量商品主图样本中提炼表达共性、分组差异和可测试主图公式。

## 分析对象

只分析主图表达，不分析完整商品页。

可以分析：

- 商品卡首图
- 店铺 Top 商品主图
- 榜单或竞品截图中的首图
- 用户按国家、平台、类目整理的图片文件夹

不要混入：

- 详情页长图
- 短视频封面
- 达人视频截图
- 品牌海报
- 不是同一商品形态的样本

## 样本表字段

建议字段：

| 字段 | 说明 |
| --- | --- |
| 编号 | S01、S02 |
| 文件/链接 | 原始路径或链接 |
| 国家/平台 | 例如 MY TikTok Shop |
| 类目/子类目 | 路径原文优先 |
| 店铺/商品 | 原文保留 |
| Top 依据 | 销量、排名、用户口述或待补 |
| 价格/销量 | 有则记录，没有写待补 |
| 主图类型 | 白底、场景、人物、对比、套装、信息图 |
| 第一视觉 | 用户第一眼看到什么 |
| 点击理由 | 让用户点进去的可能理由 |
| 风险点 | 夸大、误导、合规、侵权、规格不清 |

字段缺失时保留「待补」，不要补编。

## 清洗规则

剔除：

- 图片模糊到无法识别主体。
- 明显不是主图。
- 与样本类目差异过大。
- 重复图或同一商品多张重复截图。
- 路径和图片内容明显冲突，且无法确认。

保留但标记：

- 国家或时间未知。
- 销量或 Top 依据未知。
- 图片可读但文案不清。
- 同类目内不同子品类，需要后续分组。

## 分组方法

优先按子类目分组，其次按价格带、商品形态和使用场景分组。

常见分组：

- 标品 / 非标品
- 单件 / 套装 / 多变体
- 功效型 / 装饰型 / 工具型 / 消耗型
- 低客单 / 中客单 / 高客单
- 白底展示 / 场景展示 / 人物使用 / 效果对比

跨组共性必须谨慎，只写真正反复出现的表达事实。

## 高频共性写法

共性必须包含三件事：

```text
共性：____
证据：出现在 S__、S__、S__，表现为____。
边界：适用于____，不一定适用于____。
```

不要写：

- 「说明这样能爆」
- 「说明平台喜欢这种图」
- 「说明这个主图点击率高」

应该写：

- 「这些样本高频使用了____表达」
- 「这可能是在降低____理解成本」
- 「可以把它作为下一轮主图测试假设」

## 主图公式

公式不是模板照抄，而是可测试的表达结构。

示例结构：

```text
公式名称：{场景结果型 / 清晰规格型 / 套装价值型 / 真实使用型}
适合商品：____
画面结构：主体____ + 场景____ + 文字____
核心变量：____
验证假设：如果用户关心____，这种图可能提高____。
风险：____
```

常见公式：

- 清晰规格型：产品主体大 + 数量/尺寸/容量前置。
- 场景结果型：真实使用场景 + 使用后结果。
- 套装价值型：套装全展开 + 数量和适用对象。
- 痛点解决型：问题场景 + 产品介入。
- 信任细节型：实拍细节 + 材质/工艺/认证依据。

## 不能推断

样本归纳后必须写不能推断：

- 不能推断这些主图一定提高点击率。
- 不能推断销量由主图造成。
- 不能推断平台偏好某个视觉风格。
- 不能推断用户自己的商品照抄就有效。
- 不能跨国家、跨类目直接复用公式。

## 转成测试

把共性转成测试动作：

1. 选一个目标：点击、规格理解、加购或下单。
2. 选一个主图公式。
3. 明确只改变一个主要变量。
4. 保持价格、标题、SKU、流量来源和统计时间尽量一致。
5. 设置观察指标和最低样本量。

如果用户没有自己的页面和数据，输出「测试前准备」而不是诊断结论。
