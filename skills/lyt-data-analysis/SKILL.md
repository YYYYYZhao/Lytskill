# Lyt-data-analysis：数据分析诊断

目标：用数据和反馈找准问题，输出能执行的下一步。默认少讲概念，多给判断、证据和动作。

适用输入：业务描述、用户反馈、截图、CSV/XLSX、广告/店铺/商品/内容/交付数据。未说明行业时，按通用业务漏斗分析，不强行套 TikTok Shop。

## 核心原则

1. 先判断数据能不能回答问题，再回答问题。
2. 区分事实、推测、建议；样本越少，结论越保守。
3. 先找最大瓶颈，不平均用力。
4. 电商/投放分析必须回到：`GMV = 曝光 × 点击率 × 点击成交率 × 客单价`。
5. 不看单点指标，必须组合看曝光、点击率、点击成交率、客单价和 GMV。
6. 原始标识必须保留源数据原文，包括商品名、SKU、货号、店铺名、达人名、广告组名、素材名、订单号、国家/地区。
7. 用户结论、解释、行动项、补数清单用中文；英文只保留在原始标识、平台固定指标、通用缩写和公式里。
8. 方案必须能执行，不能只写「优化入口」「提升转化」「补点击」「补曝光」。

## 默认流程

1. 明确对象：反馈、漏斗、投放、商品、内容、交付。
2. 数据体检：看时间范围、样本量、指标口径、分组维度、缺失异常、是否有对照组。
3. 先做 GMV 四变量拆解，再判断主要瓶颈。
4. 用「现象 -> 证据 -> 假设 -> 动作」输出。
5. 数据不足时，直接给最小补数清单，不硬编完整结论。

如果用户没有说清楚目标，只追问一句：

```text
你这次最想判断的是「问题出在哪」，还是「下一步怎么优化」？如果有数据，把时间范围和核心指标一起发我。
```

## GMV 四变量诊断

口径：

- 点击率 = 点击 / 曝光。
- 点击成交率 = 成交人数、订单数或成交件数 / 点击，分母分子要跟源数据一致。
- 客单价 = GMV / 成交人数、订单数或成交件数，分母要和点击成交率口径一致。
- 如果源数据里的「转化率」是曝光到成交，先说明口径，再继续分析。

组合判断：

| 组合现象 | 优先判断 | 下一步 |
|---|---|---|
| 曝光少，点击也少 | 流量样本不足 | 先查日预算、投入产出比目标、链接历史数据、已跑时长和基础曝光；只有用户主动提到审核、受限、拒审等异常时，才查广告状态 |
| CTR 低，有成交或加购 | 前端入口偏弱，后端有信号 | 先改商品卡/素材入口，再补样本 |
| CTR 高，点击少，0 成交 | 前端信号可以，成交样本不足 | 补点击和后链路数据，不急着判详情页 |
| 点击足够，点击成交率低 | 承接偏弱 | 查详情页、价格、评价、优惠、运费、发货、规格、信任 |
| 点击成交率高，曝光低 | 有信号但没放量 | 控制成本补曝光，验证能否稳定 |
| 客单价低 | GMV 天花板低 | 做套装、组合包、加价购、阶梯优惠 |

推荐句式：

```text
这个品当前 GMV 受限于{变量}，不是单纯的{误判变量}问题。
证据是：曝光{x}，CTR{x}，点击成交率{x}，客单价{x}，GMV{x}。
所以下一步应该{动作}，而不是{容易误判的动作}。
```

## 小样本护栏

- 点击数少于 50：不能因为成交为 0，就判断商品不行、详情页差、人群不准。
- 曝光少于 500：不能稳定判断趋势，只能看初始信号。
- 判断 0 成交是否异常，先估算：`预期订单 = 点击数 × 参考点击成交率`。预期订单小于 1 时，0 成交通常是正常波动。
- 参考点击成交率优先用用户历史数据；没有历史数据时，只能用同类目经验做粗略判断，并明确是估算。
- 只有点击达到 50-100 以上，且加购、下单、支付持续为 0 或明显偏低，才优先怀疑承接问题。
- 没有加购、下单、支付数据时，不直接判断断点在详情页，把这些字段列入补数清单。

## 漏斗定位

默认漏斗：

```text
曝光 -> 点击 -> 停留/互动 -> 加购/咨询 -> 下单 -> 支付 -> 履约 -> 复购/好评
```

常用归因：

- 曝光低：先看账户/商品资格、日预算、投入产出比目标、链接历史数据、已跑时长、素材冷启动；只有用户主动提到审核、受限、拒审等异常时，才查广告状态和平台审核。
- 点击低：首图/首帧、标题前半段、价格说法、核心卖点、受众匹配。
- 加购低：详情页、评价、价格、规格、优惠、运费、配送时效、信任。
- 下单/支付低：优惠力度、结账阻力、库存、发货时效、信任不足。
- 退款高：商品预期不一致、质量、尺码、物流、客服、内容误导。
- ROI 低：客单价、毛利、点击成本、转化率、复购、退款共同作用。

## TikTok Shop GMV Max 低消耗

分析 TikTok Shop 商品成交额最大化广告，也就是 GMV Max 低消耗时，如果用户没有主动提到广告状态或异常提示，默认先按正常投放中处理，不主动追问广告状态：

- 如果用户主动说后台显示审核中、受限、拒审或其他异常提示，先查审核、拒审、受限、商品资格、账号健康，不要直接判断是预算或投入产出比问题。
- 如果用户没有主动提异常提示，优先补：日预算、投入产出比目标、选了几个产品、产品有没有可用视频素材、已跑天数、累计消耗、曝光、点击、链接历史成交/加购/点击。只有产品有可用素材且用户没有说明素材选择时，才补问素材是自己选还是系统自动选。
- GMV Max 不按传统广告字段追问「投放目标」「出价方式」「人群」「地区定向」。它的核心可控项是预算、投入产出比目标、产品选择和素材选择；产品可以多选；素材可以自己选，如果不自己选，系统会基于所选产品的可用素材自动选择。前提是所选产品有可用素材；用户已经明确没视频素材时，不要再问系统自动选素材。
- 如果所选产品没有素材，默认按商品卡/搜索入口来分析。此时重点看标题关键词、核心搜索词、商品卡自然曝光/点击和真实搜索需求；如果没有人搜，或标题没有覆盖买家会搜的词，降低投入产出比目标也不一定能起量。
- 低消耗不是直接加预算或降低投入产出比目标就能解决。先判断是商品卡/搜索入口拿不到量、链接历史数据太少、冷启动时间不够、投入产出比目标过紧，还是商品本身没有点击/成交信号；只有用户主动提异常状态时，才把审核/资格问题前置。

## CTR 判断和动作

CTR 不能孤立判断，必须对比同批商品、同商品历史、同类目/渠道基准或不同入口版本。没有基准时，只能说「需要继续对比」。

如果结论里出现「CTR 偏低」「点击入口偏弱」「点击效率不足」，下一步必须给提升 CTR 的具体动作，从下面选 2-4 个：

- 对比同批高 CTR 商品，找首图、标题、价格展示、卖点顺序差异。
- 重做首图或视频前 3 秒，让用户一眼看懂用途、结果和使用场景。
- 标题前 4-8 个词前置商品用途、核心场景或结果，不把规格词堆最前面。
- 价格说成人能看懂的话，例如「15 件装」「券后 RMxx」「平均每片 RMx」「买 2 件更便宜」「包邮」。
- 强化第一眼信任：销量、评价、真实使用图、前后对比、材质/尺寸关键信息。
- 商品卡/商品广告场景，优先做 2-3 个「商品卡新链接」；每个链接只改一个主要变量。
- 短视频场景，才说 2-3 个「视频入口素材版本」。

CTR 输出格式：

```text
CTR 问题：{当前 CTR} 低于{对照对象}。
优先动作：
1. {具体改主图/首帧/标题/价格展示的动作}
2. {具体对比或 A/B 测试动作}
验证方式：看新入口 CTR 是否高于原入口，同时点击成交率不能明显下降。
```

## 表格和商品可读性

商品分析主表默认列：

```markdown
| 商品 ID | 商品名称（短） | 曝光 | 点击 | CTR | 点击成交率 | 买家/订单 | 客单价 | GMV | 判断 | 下一步动作 |
|---|---|---:|---:|---:|---:|---:|---:|---:|---|---|
```

规则：

- 同一张表使用同一套指标列、顺序和单位，方便横向比较。
- 主表必须覆盖 GMV 四变量：曝光、CTR、点击成交率、客单价；没有字段用 `-`。
- 如果某个分组里的某些列整列都是 0、空值或 `-`，且没有信息增量，可以省略，并在表格上方说明原因。
- 低样本未成交商品默认展示曝光、点击、CTR、判断、下一步动作；点击成交率和 GMV 全为 0 时不要占主表空间。
- 商品名称（短）保留源语言原文前 24-40 个字符或前 4-8 个词，再加 `...`；不要翻译、意译、改写。
- 商品 ID、SKU、货号等定位字段完整保留。
- 如需完整名称，另做「商品名称对照表」，不要塞进核心表。

正文引用商品默认用：

```text
{商品 ID}｜{商品名称短原文}｜{中文简译}
```

要求：ID 完整，短原文保留源语言，中文简译只帮助理解商品类型；不能替代源数据字段。无法可靠翻译时写「中文简译待确认」。

## 行动标准

「立刻做」里的每个动作至少包含 5 个要素：

```text
对象：改哪个商品/链接/素材/广告组。
动作：具体做什么。
改法：首图、标题、价格、素材、预算、人群、达人、短视频等怎么改。
验证：看哪个指标，跑到多少样本。
条件：什么情况下继续、暂停或换方案。
```

补曝光/补点击必须说明方式：

- 广告数据：用小额广告补样本，曝光和点击更可控。
- 自然短视频数据：再发短视频挂车补样本。
- 达人数据：补达人素材或达人分发。
- 广告和短视频同时做时，两条数据分开看，不混在一起判断同一个版本。

表达模板：

```text
用小额广告给这个商品卡新链接补 500-1000 曝光，预算先控制在{金额/天}，不先加总预算；跑完看 CTR 是否高于原链接，点击成交率是否没有下降。
```

```text
先把预算从低 CTR 老链接挪到 CTR 更高的新链接，跑到 50 点击；如果 50 点击内还能出单或有加购，再补到 100-200 点击。没有加购和成交，就暂停这个链接。
```

指标和动作必须匹配：

| 弱变量 | 动作方向 |
|---|---|
| 曝光低 | 先查日预算、投入产出比目标、链接历史数据、已跑时长和基础曝光；用户主动提异常时，再查广告状态和审核 |
| CTR 低 | 做商品卡新链接或新素材，改首图/首帧/标题前半段/价格说法/卖点顺序 |
| 点击成交率低 | 查详情页、评价、价格、优惠、运费、发货、规格、信任 |
| 客单价低 | 做套装、组合包、加价购、阶梯优惠 |
| GMV 低 | 先拆四变量，不直接给单点动作 |

## 数据处理规则

表格数据：

1. 先读取表头和前几行，理解字段含义。
2. 做基础统计、分组对比、异常检查。
3. 优先用表格处理能力或本地脚本读取，不手工猜列。
4. 不改原始文件；如需清洗版或报告，另存新文件。
5. 输出结论标明数据日期和文件来源。
6. 原始标识字段值不翻译；指标列名可以中文化。

文字反馈：

1. 拆成对象、场景、问题、频次、影响。
2. 归类重复反馈。
3. 标出高频问题和高伤害问题。
4. 区分真实用户反馈和用户自己的解释。

## 禁止事项

- 不跳过数据体检。
- 不把偶然波动说成趋势。
- 不在点击少于 50 时，因为 0 成交就断言商品、详情页或人群有问题。
- 不在预期订单小于 1 时，把 0 成交当异常证据。
- 不因为有成交，就忽略 CTR 偏低。
- 不只看单点指标，必须回到 GMV 四变量组合判断。
- 不把超长商品名塞进核心表。
- 不强行展示没有信息增量的全 0 列。
- 不只写商品 ID，正文里至少带短原文和中文简译。
- 不中英混杂；可翻译的英文词用中文。
- 不写抽象动作；「补曝光」「补点击」必须说清楚用广告、短视频、达人还是组合方式。
- 不在没有利润、成本、退款数据时断言赚钱。
- 不把 TikTok Shop GMV Max 当传统广告分析；低消耗时不要默认追问投放目标、出价方式、人群或地区定向，也不要在用户未主动提异常时追问广告状态。先看日预算、投入产出比目标、产品选择、产品是否有可用视频素材、已跑天数、实际消耗、曝光、点击和链接历史数据。
- GMV Max 无素材且商品卡没有流量时，不要只归因为投入产出比目标偏紧；必须先判断商品卡/搜索入口，尤其是标题关键词、核心搜索词和真实搜索需求。
