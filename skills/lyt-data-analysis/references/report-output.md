# 报告与表格输出

本文件用于完整复盘、交付报告、批量分析表、对外版本和需要沉淀给团队/学员的分析结果。

## 什么时候用完整报告

只在以下情况进入完整报告：

- 用户明确说要报告、复盘、打包、给团队/学员看。
- 数据包含多商品、多广告、多日期、多渠道或利润字段。
- 结论需要支持决策，不只是回答一个小问题。
- 用户需要可复制的表格、行动清单或后续跟踪。

简单追问或单个判断不要强行写长报告。

## 报告结构

```markdown
# 数据复盘：{对象与时间范围}

## 结论先说
{最重要判断 + 当前决策}

## 数据范围与口径
{来源、时间、样本、字段、缺失、限制}

## 关键发现
1. {发现 + 证据 + 影响}
2. {发现 + 证据 + 影响}
3. {发现 + 证据 + 影响}

## 分析拆解
{GMV 四变量 / 漏斗 / 广告 / 利润 / 竞品结构}

## 行动方案
| 优先级 | 对象 | 动作 | 验证指标 | 继续/暂停条件 |
| --- | --- | --- | --- | --- |

## 风险与限制
{数据不能证明什么}

## 需要补充的数据
{只写会改变结论的字段}
```

## 表格规则

- 同一张表使用统一指标、顺序和单位。
- 商品名保留源语言短原文，商品 ID、SKU 和货号完整保留。
- 正文引用商品使用：`商品 ID｜商品名称短原文｜中文说明`。
- 全空或没有信息增量的列可以省略，但要说明。
- 分组后先给组结论，再展开重点对象。

## 行动标准

每个动作至少写清：

```text
对象：改哪个商品、链接、素材、广告组或渠道。
动作：具体做什么。
改法：主图、标题、价格、素材、预算、达人或详情怎么改。
验证：看哪个指标和样本。
条件：什么情况下继续、暂停或换方案。
```

没有验证条件的动作，不写进最终行动方案。
