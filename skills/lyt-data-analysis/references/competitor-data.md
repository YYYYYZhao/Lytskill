# 竞品与第三方工具数据

本文件用于竞品、FastMoss 类工具、公开榜单、渠道结构、价格带、达人结构和第三方页面分析。

## 基本顺序

```text
明确问题 -> 选择资源条件相似的竞品 -> 提取数据 -> 区分事实/推断/假设 -> 设计验证
```

竞品不是答案，只是参照。

## 数据口径

| 数据 | 正确用法 | 常见误判 |
| --- | --- | --- |
| 销量 | 判断售出件数 | 直接当订单数 |
| 销售额 | 判断成交规模 | 当利润 |
| 商品数占比 | 看上架结构 | 当成交结构 |
| 销量/销售额占比 | 看成交主力 | 忽略价格带 |
| 渠道占比 | 判断成交方式 | 推断单一打法 |
| 广告标识 | 说明可能有投放迹象 | 推出广告费和 ROI |
| 达人数量 | 看达人池规模 | 以为越多越好 |
| 达人销售额 | 看渠道贡献 | 直接得出达人利润 |

第三方工具页面口径不同，不直接相加；看到的数据只能证明页面展示的内容。

## 竞品选择

优先选择：

- 同国家/平台
- 同子类目
- 相近价格带
- 相近履约条件
- 相近流量渠道
- 资源条件不要远超用户太多

不要只选头部店铺。头部可以看天花板，但不一定能当用户当前阶段对标。

## 分析维度

| 维度 | 看什么 |
| --- | --- |
| 商品结构 | 主力品、长尾品、价格带、SKU |
| 流量结构 | 商品卡、短视频、直播、达人、广告 |
| 表达结构 | 主图、标题、卖点、场景、详情页 |
| 信任结构 | 评价、销量、店铺、物流、售后 |
| 供给结构 | 价格、包装、重量、履约难度 |
| 竞争壁垒 | 供应链、内容、达人池、评价沉淀、广告能力 |

## 输出要求

竞品结论必须写：

- 这是公开数据还是用户后台数据。
- 哪些是事实。
- 哪些只是推断。
- 这个竞品和用户是否资源相似。
- 哪些做法可以参考，哪些不能照抄。
- 下一步用什么测试验证。

## 禁止事项

- 不把竞品销量当成利润。
- 不把竞品广告标识当成广告投入产出。
- 不把头部结果当作新手路径。
- 不把第三方工具截图当作平台后台事实。
- 不因为竞品能卖，就判断用户一定能卖。
