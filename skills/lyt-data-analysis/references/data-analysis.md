## 数据分析总原则

数据分析不是把后台数字解释一遍，而是把可见事实、合理推理和不能确定的假设分开，最后回到可执行动作。

核心顺序：

```text
明确问题 -> 数据体检 -> 口径校验 -> 拆变量 -> 找最大瓶颈 -> 分清事实/推理/假设 -> 输出动作
```

基本原则：

- 数据不是结论，数据是验证假设的工具。
- 不看单点指标，必须组合判断。
- 不把相关性当因果。
- 不把工具截图当完整事实。
- 不把销量当订单数，不把 GMV 当利润，不把广告 ROI 当真实利润。
- 样本越少，结论越保守。
- 所有动作必须能落到商品、链接、素材、广告组、达人、价格、物流或详情页。

## 数据体检

分析前先判断数据能不能回答问题：

| 检查项 | 要看什么 | 常见风险 |
| --- | --- | --- |
| 时间范围 | 几天，是否覆盖变化前后 | 把一天波动当趋势 |
| 样本量 | 曝光、点击、下单、订单是否足够 | 小样本 0 成交被误判 |
| 字段口径 | 点击率、转化率、订单、销量、GMV 的定义 | 不同页面字段混算 |
| 分组维度 | 商品、广告组、素材、渠道、达人、日期是否可拆 | 全店数据看不出商品级动作 |
| 缺失异常 | 是否缺加购、下单、成本、退款、利润 | 只能判断流量，不能判断利润 |
| 对照组 | 历史、同批商品、竞品、渠道、素材版本 | 没基准却判断好坏 |

数据不足时，不硬编完整结论，只给「初步判断」和最小补数清单。

## 口径校验

字段没确认，不做结论。

| 易错字段 | 正确处理 |
| --- | --- |
| 销量 | 通常是售出商品件数，不一定等于订单数 |
| 订单数 | 用户下单次数；一个订单可能包含多件商品 |
| 客单价 | 成交额 / 订单数；没有订单数时不能精确计算 |
| 平均单件成交价 | 成交额 / 销量；不能替代客单价 |
| GMV | 成交额，不等于到账金额，也不等于利润 |
| ROI | 广告收入效率，不等于真实经营利润 |
| 广告归因 GMV | 可能包含自然、达人或系统归因订单，不能直接等同广告真实增量 |
| 总趋势页 GMV | 用来判断总盘子 |
| 视频页/达人页 GMV | 用来判断打法和渠道，不要和总趋势页直接叠加 |
| 商品价格范围 | 不代表主力成交价格，优先看成交价格带 |

## 事实、推理、假设、动作

竞品截图、FastMoss 数据、后台截图和用户口述都要分层。

| 类型 | 定义 | 写法 |
| --- | --- | --- |
| 事实 | 数据里直接看得到 | 「近 7 天曝光 1200，点击 24，点击率 2%」 |
| 推理 | 基于事实的合理判断 | 「点击率不算完全无效，但点击样本还小」 |
| 假设 | 还不能确认，需要验证 | 「可能是详情页信任不足，需要看加购和下单」 |
| 动作 | 可执行验证方案 | 「先跑到 50 点击，再看加购和下单」 |

不要把「推理」写成事实，也不要把「假设」写成结论。

## GMV 四变量诊断

核心公式：

```text
GMV = 曝光 × 点击率 × 点击成交率 × 客单价
```

口径：

- 点击率 = 点击 / 曝光。
- 点击成交率 = 成交人数、订单数或成交件数 / 点击，分母分子要跟源数据一致。
- 客单价 = GMV / 订单数；没有订单数时，用平均单件成交价替代时必须说明。
- 如果源数据里的「转化率」是曝光到成交，先说明口径，再继续分析。

组合判断：

| 组合现象 | 优先判断 | 下一步 |
| --- | --- | --- |
| 曝光少，点击也少 | 流量样本不足 | 先查渠道、预算、投入产出比目标、链接历史数据、已跑时长和基础曝光 |
| 点击率低，有加购或成交 | 前端入口偏弱，后端有信号 | 先改商品卡/素材入口，再补样本 |
| 点击率高，点击少，0 成交 | 前端信号可以，成交样本不足 | 补点击和后链路数据，不急着判详情页 |
| 点击足够，点击成交率低 | 承接偏弱 | 查详情页、价格、评价、优惠、运费、发货、规格、信任 |
| 点击成交率高，曝光低 | 有信号但没放量 | 控制成本补曝光，验证能否稳定 |
| 客单价低 | GMV 天花板低 | 做套装、组合包、加价购、阶梯优惠 |
| GMV 高但利润薄 | 收入好看但成本吞利润 | 查货品成本、物流、平台费、达人佣金、广告费、优惠、退款 |

推荐句式：

```text
这个品当前 GMV 受限于{变量}，不是单纯的{误判变量}问题。
证据是：曝光{x}，点击率{x}，点击成交率{x}，客单价{x}，GMV{x}。
所以下一步应该{动作}，而不是{容易误判的动作}。
```

## 小样本护栏

- 点击数少于 50：不能因为成交为 0，就判断商品不行、详情页差、人群不准。
- 曝光少于 500：不能稳定判断趋势，只能看初始信号。
- 判断 0 成交是否异常，先估算：`预期订单 = 点击数 × 参考点击成交率`。
- 预期订单小于 1 时，0 成交通常是正常波动。
- 参考点击成交率优先用用户历史数据；没有历史数据时，只能用同类目经验做粗略判断，并明确是估算。
- 只有点击达到 50-100 以上，且加购、下单、成交持续为 0 或明显偏低，才优先怀疑承接问题。
- 没有加购、下单、成交数据时，不直接判断断点在详情页，把这些字段列入补数清单。

## 漏斗定位

默认漏斗：

```text
曝光 -> 点击 -> 停留/互动 -> 加购/咨询 -> 下单 -> 成交 -> 履约 -> 复购/好评
```

常用归因：

| 断点 | 常见原因 |
| --- | --- |
| 曝光低 | 账户/商品资格、渠道权重、日预算、投入产出比目标、链接历史数据、已跑时长、素材冷启动 |
| 点击低 | 首图/首帧、标题前半段、价格说法、核心卖点、受众匹配 |
| 加购低 | 详情页、评价、价格、规格、优惠、运费、配送时效、信任 |
| 下单低 | 优惠力度、库存、发货时效、信任不足、价格对比 |
| 复购/好评低 | 商品预期不一致、质量、尺码、物流、客服、内容误导 |
| ROI 低 | 客单价、毛利、点击成本、点击成交率、复购、退款共同作用 |

## 商品卡、短视频、直播场景

不同场景不能混在一起分析。

| 场景 | 关键链路 | 核心指标 | 常见动作 |
| --- | --- | --- | --- |
| 商品卡/搜索 | 搜索 -> 曝光 -> 点击 -> 加购 -> 下单 -> 成交 | 曝光、点击率、加购率、点击成交率、搜索词 | 标题关键词、主图、价格、评价、物流、商品卡新链接 |
| 短视频/推荐 | 推荐 -> 观看 -> 点击商品 -> 加购 -> 下单 -> 成交 | 播放、完播/互动、商品点击率、加购、下单 | 前 3 秒、场景、痛点、引导点击、详情页承接 |
| 直播 | 推荐 -> 进房 -> 停留 -> 点击商品 -> 下单 -> 成交 | 进房、停留、商品点击、讲品转化、场均销售额 | 讲品顺序、优惠、爆品反复讲、直播排班 |
| 达人 | 达人内容 -> 点击商品 -> 加购 -> 下单 -> 成交 | 达人数、达人层级、达人类型、销售额占比 | 分层达人池、垂类达人、头部爆发、中腰部成交 |

视频爆但不出单时：

- 播放量高只说明内容有传播性，不等于购买意愿高。
- 先看播放到商品页点击率，再看点击到加购、加购到下单。
- 评论问「多少钱/怎么买」说明有需求但可能信任或购买路径弱。
- 评论问「这是什么/有什么用」说明内容和产品卖点没有讲清楚。

## GMV Max 和广告诊断

分析 TikTok Shop 商品成交额最大化广告，也就是 GMV Max 时，不把它当传统广告。

GMV Max 是商品增长调度器，受这些因素影响：

```text
商品基础盘 -> 标题/主图/价格/评价/库存/物流 -> 可用素材 -> 系统调度 -> 曝光 -> 点击 -> 加购 -> 下单 -> 成交 -> ROI
```

如果用户没有主动提广告状态或异常提示，默认先按正常投放中处理，不主动追问广告状态。

GMV Max 不按传统广告字段追问「投放目标」「出价方式」「人群」「地区定向」。

核心字段：

| 字段 | 用途 |
| --- | --- |
| 日预算 | 判断预算是否限制探索 |
| 实际消耗 | 判断广告有没有跑出去 |
| 预算消耗率 | 判断是否跑满预算 |
| 投入产出比目标 | 判断约束是否过紧 |
| 实际 ROI | 判断广告收入效率 |
| 保本 ROI | 判断是否覆盖成本 |
| 订单数 | 判断 ROI 是否稳定 |
| 平均下单成本 | 判断成交成本是否可接受 |
| 产品数 | 判断是否全店混跑、是否要拆产品 |
| 素材数量/可用性 | 判断系统有没有内容资产 |
| 链接历史数据 | 判断系统是否有点击和成交信号 |

预算消耗率：

```text
预算消耗率 = 实际消耗 / 日预算
```

| 预算消耗率 | 判断 | 下一步 |
| --- | --- | --- |
| 低于 50% | 广告没跑出去 | 查投入产出比目标、商品基础盘、素材、链接历史数据 |
| 50%-80% | 有探索机会 | 看订单、点击成本、产品集中度 |
| 高于 80% | 接近预算上限 | 看 ROI、订单稳定性、保本 ROI |
| 接近 100% | 预算可能限制探索 | ROI 达标且订单稳定时，再考虑小幅加预算 |

保本 ROI：

```text
保本 ROI = 1 / 毛利率
```

例子：

```text
毛利率 50%，保本 ROI = 2
毛利率 25%，保本 ROI = 4
```

注意：毛利率必须扣除商品成本、物流、平台费、佣金、退款等关键成本后再估。后台 ROI 高，不代表真实利润一定高。

广告组合判断：

| 现象 | 判断 | 动作 |
| --- | --- | --- |
| 消耗低，ROI 高 | 样本少，可能是目标太紧或探索机会少 | 看保本 ROI，适当放宽目标，补素材 |
| 消耗低，ROI 正常 | 跑不满预算，曝光少 | 查商品基础盘、素材、标题关键词、链接历史数据 |
| 消耗高，ROI 达标 | 有放量机会 | 预算消耗率高于 80%、订单稳定时，小幅加预算 |
| 消耗高，ROI 低 | 能跑但承接差或成本高 | 优化详情页、价格、评价、素材，或换产品 |
| ROI 高，GMV 小 | 效率看似好，但规模小 | 放宽目标、补素材、补产品历史信号 |
| GMV 高，利润薄 | 收入高但真实成本吃掉利润 | 提高目标、优化供应链/物流/价格，或换品 |
| 点击多，成交少 | 承接弱 | 查详情页、价格、物流、评价、优惠 |
| 单子集中少数产品 | 系统在赛马筛商品 | 拆计划，保留强产品，移除弱产品 |
| 全店跑看不清 | 商品级动作不明确 | 按类目/产品组拆分测试 |

什么时候加预算：

- 预算消耗率达到 80% 以上。
- 实际 ROI 高于保本 ROI。
- 订单稳定，不是 1-2 单拉高 ROI。
- 平均下单成本可接受。
- 评分、退款、差评没有恶化。

什么时候放宽投入产出比目标：

- 广告跑不出去。
- 冷启动或探索人群阶段。
- ROI 高但 GMV 小、消耗少。

什么时候减少产品：

- 全店跑，看不到商品级动作。
- 类目太杂，判断不了哪个产品有机会。
- 少数产品贡献大部分订单，其余产品只消耗不出单。

## 成本、利润和保本投入产出比

GMV 不是利润。经营利润至少要扣：

```text
经营利润 = GMV - 货品成本 - 物流仓储 - 平台费用 - 达人佣金 - 广告费 - 优惠补贴 - 样品成本 - 人工运营 - 售后损耗
```

完整成本项：

| 成本项 | 说明 |
| --- | --- |
| 货品成本 | 采购价、包装、质检 |
| 物流仓储 | 国内段、跨境段、本地尾程、仓储处理 |
| 平台费用 | 类目佣金、交易手续费、平台支持费、活动服务费 |
| 达人佣金 | 标准佣金、广告佣金、样品成本、额外合作费 |
| 广告费 | 实际广告消耗，不从竞品工具里猜死 |
| 优惠补贴 | 店铺券、活动价、免邮补贴、平台活动 |
| 售后损耗 | 退款、退货、差评、丢件、重新发货 |
| 人工运营 | 主播、剪辑、客服、达人 BD、投手 |

利润分析规则：

- 没有成本、退款、平台费、佣金、物流、广告费时，不断言赚钱。
- 如果只有 GMV 和销量，只能算平均单件成交价，不能算真实客单价。
- 平台支持费、订单处理费这类按订单收费的项目，不能直接用销量相乘。
- 没有订单数时，必须做 1 件/单、1.5 件/单、2 件/单等区间假设。
- 成本测算要写清楚数据来源和假设，不要装成确定事实。

## 竞品和 FastMoss 类数据

竞品分析不是学习竞品，而是用外部数据验证假设。

正确顺序：

```text
明确问题 -> 找资源条件相似的竞品 -> 提取数据 -> 分清事实/推理/假设 -> 小批量验证
```

FastMoss 类数据常用判断：

| 数据 | 正确用法 | 常见误判 |
| --- | --- | --- |
| 时间范围 | 所有结论围绕同一个周期 | 把历史累计和近 28 天混算 |
| 销量 | 判断售出商品件数 | 直接当订单数 |
| 销售额 | 判断成交规模 | 当利润 |
| 平均单件成交价 | 销售额 / 销量 | 当客单价 |
| 商品数占比 | 看上架结构 | 当成交结构 |
| 销量/销售额占比 | 看真正成交主力 | 忽略价格带和类目集中 |
| 前 5 商品占比 | 判断爆品集中度 | 以为每个品都平均卖 |
| 渠道占比 | 判断成交方式 | 只说单一渠道 |
| AD 标识 | 说明可能有投放迹象 | 推出广告费、广告 ROI |
| 达人数量 | 看达人池规模 | 以为达人越多越好 |
| 达人销售额 | 看达人效率 | 忽略粉丝层级和达人类型 |

竞品拆解输出表：

```markdown
| 模块 | 可见事实 | 合理推理 | 不能确定 | 下一步验证 |
| --- | --- | --- | --- | --- |
```

重要护栏：

- 数据页面口径不同，不直接相加。
- 成交价格带看销量和销售额，不只看商品数或最低价。
- 成本测算看主力成交价格带，不看标价范围。
- 看到 AD 标识，只能说有投放迹象，不能说广告费是多少。
- 看到达人 GMV，只能判断达人渠道贡献，不能直接得出达人利润。
- 竞品成功是时间、地点、资源、供应链、达人、内容和运气的组合，不能直接复制。

## 数据误导

常见误导：

| 误导 | 解释 | 应对 |
| --- | --- | --- |
| 数据涨了就是做对了 | 可能是活动、平台溢出流量、偶然波动 | 看对照组和多天趋势 |
| 单点数据好就是整体好 | 粉丝、播放、GMV 可能和利润无关 | 回到业务目标 |
| ROI 高就是广告好 | 订单少会拉高 ROI，且不代表利润 | 看订单稳定性和保本 ROI |
| GMV 高就能放大 | GMV 可能被成本、退款、佣金吃掉 | 做利润拆解 |
| 竞品数据给了答案 | 竞品只提供外部线索 | 用小批量测试验证 |
| 相关性就是因果 | 看到 A 和 B 一起出现，不代表 A 导致 B | 设计对照和 AB 测试 |

```markdown
| 广告/产品 | 日预算 | 实际消耗 | 预算消耗率 | 曝光 | 点击 | 下单 | 成交额 | 实际 ROI | 保本 ROI | 平均下单成本 | 判断 | 动作 |
| --- | ---: | ---: | ---: | ---: | ---: | ---: | ---: | ---: | ---: | ---: | --- | --- |
```

规则：

- 同一张表使用同一套指标列、顺序和单位，方便横向比较。
- 主表必须覆盖 GMV 四变量：曝光、点击率、点击成交率、客单价；没有字段用 `-`。
- 如果某个分组里的某些列整列都是 0、空值或 `-`，且没有信息增量，可以省略，并在表格上方说明原因。
- 低样本未成交商品默认展示曝光、点击、点击率、判断、下一步动作；点击成交率和 GMV 全为 0 时不要占主表空间。
- 商品名称（短）保留源语言原文前 24-40 个字符或前 4-8 个词，再加 `...`；不要翻译、意译、改写。
- 商品 ID、SKU、货号等定位字段完整保留。
- 正文引用商品默认用：`{商品 ID}｜{商品名称短原文}｜{中文简译}`。

## 行动标准

「立刻做」里的每个动作至少包含 5 个要素：

```text
对象：改哪个商品/链接/素材/广告组。
动作：具体做什么。
改法：首图、标题、价格、素材、预算、达人、短视频等怎么改。
验证：看哪个指标，跑到多少样本。
条件：什么情况下继续、暂停或换方案。
```

补曝光/补点击必须说明方式：

- 广告数据：用小额广告补样本，曝光和点击更可控。
- 自然短视频数据：再发短视频挂车补样本。
- 达人数据：补达人素材或达人分发。
- 商品卡数据：做商品卡新链接或调整标题/主图/价格后分开看。
- 广告和短视频同时做时，两条数据分开看，不混在一起判断同一个版本。

指标和动作必须匹配：

| 弱变量 | 动作方向 |
| --- | --- |
| 曝光低 | 查渠道、预算、投入产出比目标、链接历史数据、商品资格、素材 |
| 点击率低 | 改首图/首帧/标题前半段/价格说法/卖点顺序 |
| 加购低 | 查详情页、规格、评价、价格、运费、购买理由 |
| 点击成交率低 | 查详情页、评价、价格、优惠、运费、发货、规格、信任 |
| 客单价低 | 做套装、组合包、加价购、阶梯优惠 |
| ROI 低 | 先看保本 ROI，再查成交成本、毛利、退款和承接 |
| GMV 低 | 先拆四变量，不直接给单点动作 |

## 数据处理规则

表格数据：

1. 先读取表头和前几行，理解字段含义。
2. 做基础统计、分组对比、异常检查。
3. 优先用表格处理能力或本地脚本读取，不手工猜列。
4. 不改原始文件；如需清洗版或报告，另存新文件。
5. 输出结论标明数据日期和文件来源。
6. 原始标识字段值不翻译；指标列名可以中文化。

文字反馈：

1. 拆成对象、场景、问题、频次、影响。
2. 归类重复反馈。
3. 标出高频问题和高伤害问题。
4. 区分真实用户反馈和用户自己的解释。

截图和竞品工具：

1. 先描述能直接看见的字段和时间范围。
2. 再写推理链。
3. 最后写不能确定的部分。
4. 不把截图外没有的数据当结论。
