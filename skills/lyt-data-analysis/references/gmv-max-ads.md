# GMV Max 与广告分析

本文件用于 GMV Max、广告低消耗、预算消耗率、ROI、保本 ROI、平均下单成本和广告放量判断。

## 核心链路

GMV Max 是商品增长调度器，不能照搬传统广告字段。

```text
商品基础盘 -> 标题/主图/价格/评价/库存/物流 -> 素材 -> 系统调度 -> 曝光 -> 点击 -> 加购 -> 下单 -> 成交 -> ROI
```

核心字段：

| 字段 | 用途 |
| --- | --- |
| 日预算、实际消耗、预算消耗率 | 判断能否跑出探索 |
| 投入产出比目标、实际 ROI、保本 ROI | 判断效率约束 |
| 订单数、平均下单成本 | 判断稳定性和成交成本 |
| 产品数、素材数 | 判断系统可调度空间 |
| 链接历史数据 | 判断是否已有点击和成交信号 |

```text
预算消耗率 = 实际消耗 / 日预算
保本 ROI = 1 / 毛利率
```

## 现象判断

| 现象 | 判断方向 |
| --- | --- |
| 消耗低，ROI 高 | 样本少、目标偏紧或探索机会少 |
| 消耗低，ROI 正常 | 查商品基础盘、素材和历史信号 |
| 消耗高，ROI 达标 | 订单稳定时考虑逐步放量 |
| 消耗高，ROI 低 | 查承接、成本、产品和素材 |
| ROI 高，GMV 小 | 效率好看但规模小 |
| 单子集中少数产品 | 系统赛马，拆产品看贡献 |

## 低消耗优先排查

1. 已跑时长是否足够，是否只是刚建计划。
2. 预算消耗率是否低于 50%。
3. 投入产出比目标是否明显高于保本 ROI。
4. 产品是否太少，或链接没有历史点击/加购/成交。
5. 是否缺少可用素材。
6. 商品卡自然流量是否也没有曝光和点击。
7. 是否有商品状态、库存、物流、类目或合规问题。

不要把「降低投入产出比目标」写成唯一动作。

## 放量判断

满足以下条件才考虑逐步放量：

- 消耗稳定，不是只跑出零星样本。
- 订单数足够，不只是 1-2 单偶然成交。
- 实际 ROI 接近或高于保本 ROI。
- 平均下单成本在毛利可承受范围内。
- 退款、差评、物流和履约没有明显恶化。

放量动作要写清：加预算幅度、观察时间、回撤条件和下一轮判断指标。
