# 最小验证与实验设计规则

本文件用于 lyt 系列输出「怎么测、看什么指标、什么条件继续/暂停」时读取。纯路由、纯学习、纯机制讲解或只做问题追问时不用读取。

## 什么时候用

用户出现以下需求时使用：

- 怎么测试这个产品、页面、主图、价格、素材或广告。
- 要不要继续、暂停、放量、回滚或换方案。
- A/B 测试、单变量验证、改版验证、MVP 测试。
- 输出行动方案时需要写验证指标和继续/暂停条件。

## 基本原则

1. 先明确要验证的假设，不把动作本身当目标。
2. 每轮只改一个主要变量；多变量同时改时，只能验证整体方案，不能拆因果。
3. 样本不足时不下强结论，先写最低观察样本。
4. 指标必须和动作对应，不能用下单指标验证一个只影响点击的改动。
5. 继续、暂停、回滚条件必须提前写清，不事后解释。
6. 测试不能绕开合规、成本、履约和平台规则风险。

## 验证卡

默认使用这张卡：

```text
假设：____
对象：商品 / 页面 / 主图 / 标题 / 价格 / 素材 / 广告 / 达人
改动变量：____
保持不变：价格 / 标题 / SKU / 主图 / 详情页 / 流量来源 / 预算 / 统计时间
观察指标：____
最低样本：____
观察周期：____
继续条件：____
暂停条件：____
回滚或下一轮：____
```

信息少时压缩成：

```text
本轮只验证：____
看哪个指标：____
达到什么继续：____
低于什么暂停：____
```

## 指标对应

| 要验证什么 | 优先指标 | 不要只看 |
| --- | --- | --- |
| 主图/标题/商品卡表达 | 曝光、点击、点击率 | 下单 |
| 详情页/规格/信任 | 点击、加购、下单、点击到下单 | 曝光 |
| 价格/优惠/运费 | 加购、下单、客单价、利润 | 点击率 |
| 短视频素材 | 播放、完播、商品点击、加购、下单 | 播放量 |
| GMV Max / 广告 | 消耗、曝光、点击、下单、ROI、平均下单成本 | ROI 单点 |
| 产品是否继续测 | 曝光、点击、加购、下单、成本、风险 | 竞品销量 |
| 利润策略 | 毛利、保本 ROI、退款、物流、广告费 | GMV |

## 样本护栏

- 曝光少于 500：只能看初始曝光和点击信号。
- 点击少于 50：不能因为 0 下单判断商品、详情页或人群有问题。
- 只有 1-2 单：不能判断稳定成交或稳定 ROI。
- 只有单日数据：先看是否有活动、断货、改价、异常流量或平台波动。
- 没有修改前后对照：不能把变化归因到某个改动。

样本门槛不是绝对规则；如果用户有历史基线或同批对照，优先用用户自己的基线。

## 单变量规则

推荐写法：

```text
本轮只改____，其他保持不变。这样如果____变化，才更可能判断它和____有关。
```

如果必须同时改多个变量：

```text
本轮同时改了____，所以只能验证整体方案是否更好，不能判断到底是哪个变量起作用。
```

## 继续与暂停

继续条件要正向、可观察：

- 达到最低样本后点击率高于同批基线。
- 有连续加购或下单。
- ROI 接近或高于保本 ROI，且订单不是单个偶然样本。
- 退款、差评和物流风险没有恶化。

暂停条件要具体：

- 达到最低样本后仍没有点击或加购信号。
- 关键成本补齐后保本 ROI 明显不可达。
- 合规、侵权、认证、履约风险无法排除。
- 多轮测试后同一断点没有改善。

不要写「看情况继续」。
