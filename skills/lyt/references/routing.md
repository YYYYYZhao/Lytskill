# lyt 路由知识包

这个文件是 `lyt` 主路由的随包参考。它用于处理模糊问题、多个意图同时出现、或不确定该调用哪个 lyt 子 skill 的情况。

## 路由目标

| 子 skill | 核心职责 | 不负责什么 |
| --- | --- | --- |
| `lyt-product-selection` | 选品、候选品筛选、单品是否值得测、类目拆解、供应链/利润/履约/合规初筛 | 不做完整广告诊断，不分析大表格里的经营变量 |
| `lyt-traffic-logic` | 讲清流量分配逻辑，尤其是商品卡/搜索、短视频/推荐、广告/GMV Max、冷启动 | 不做具体店铺或商品诊断，不追问补数 |
| `lyt-problem-clarifier` | 用户只给模糊问题时，先追问动作、数据断点、关键证据 | 不直接给完整诊断，不处理大表格分析 |
| `lyt-data-analysis` | 用户给了截图、表格、指标、后台数据时，做数据体检和变量诊断 | 不在数据不足时硬下结论，不做纯知识讲解 |

## 常见分流

| 用户说法 | 优先路由 | 原因 |
| --- | --- | --- |
| 「我不知道卖什么」 | `lyt-product-selection` | 用户缺选品路径 |
| 「这个产品能不能做」 | `lyt-product-selection` | 用户要判断商品是否值得测 |
| 「为什么店铺没流量」 | `lyt-traffic-logic` 或 `lyt-problem-clarifier` | 如果用户想理解逻辑，走流量逻辑；如果要定位自己店铺，走问题追问 |
| 「平台为什么不给我流量」 | `lyt-traffic-logic` | 先补平台分配逻辑 |
| 「为什么现在不出单」 | `lyt-problem-clarifier` | 先追问动作和数据断点 |
| 「广告低消耗」 | `lyt-problem-clarifier` | 先追问 GMV Max 相关最小必要信息 |
| 「这是我的表格/截图，帮我看问题」 | `lyt-data-analysis` | 已有数据，先体检再分析 |

## 易混场景

| 场景 | 判断方式 |
| --- | --- |
| 用户问「没流量」，但没有给数据 | 先判断他是在学习逻辑还是诊断问题；学习走 `lyt-traffic-logic`，诊断走 `lyt-problem-clarifier` |
| 用户问「不出单」，并给了完整曝光、点击、加购、成交数据 | 直接走 `lyt-data-analysis` |
| 用户问「广告跑不出去」但没数据 | 走 `lyt-problem-clarifier`，按 GMV Max 低消耗追问 |
| 用户问「某个类目有没有机会」 | 走 `lyt-product-selection` |
| 用户问「母猪的产后护理」这类无关问题 | 拒绝，不路由 |

## 总原则

路由只做分流，不做完整回答。命中后必须继续执行目标 skill。
